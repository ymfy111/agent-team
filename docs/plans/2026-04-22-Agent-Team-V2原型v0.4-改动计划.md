## 一、整体策略

只做三类改动：
1. **结构纠偏** — 修复与设计文档不一致的信息架构
2. **核心状态补齐** — 补齐设计模型中已定义但原型缺失的字段/状态
3. **命名/视觉收敛** — 统一文案与视觉权重

**不做**：新页面、新指标、新交互层、骨架屏精修、抽屉扩展为详情页。

---

## 二、改动清单（12 项）

### A. 结构纠偏（P0）

#### A1. 新增默认总览页 `[P0]`
- 位置：左侧菜单顶部，作为默认首页
- 包含：
  - **健康度卡片**：Master 数 / Worker 数 / 待决策数 / Team 数（healthy/degraded 分色）
  - **全局拓扑图**：SVG 渲染，节点表示 Team/Worker，连线表示绑定关系
  - **最近事件流**：最多 20 条，含时间戳、事件类型、关联 Team/Worker
- 进入面板默认打开此页

#### A2. 重做 Team 详情页布局 `[P0]`
- 改为左 35% / 右 65% 双栏布局
- **左侧 35%**（垂直分组）：
  - Master 状态卡（在线时长、当前任务、sessionId、endpoint）
  - 已加入成员（worker 列表，含状态/角色/心跳/currentTaskSummary）
  - 可加入 Worker（unclaimed 池筛选当前可加入的）
  - 其他 Team 占用 Worker（只读展示，含归属 Team 名）
- **右侧 65%**：上下文操作区
  - 选中 worker → 显示详细信息 + 派任务/移出/查看任务
  - 选中决策 → 显示 Markdown 上下文 + 选项按钮 + 驳回
  - 默认显示 Team 最近活动流

#### A3. 删除"+ 新建 Team"按钮 `[P0]`
- 原因：Team 由 master 注册时自动创建，手工新建破坏 1:1 绑定语义
- 替换为：空态时显示引导文字"Team 由主智能体注册自动创建。请先启动主智能体。"

#### A4. 系统设置页填充最小内容 `[P0-部分]`
- 改为只读信息卡，不做配置编辑
- 字段：
  - 注册中心版本
  - 运行时长（uptime）
  - 监听端口
  - 当前连接数
  - 心跳超时阈值（只读）
  - 持久化文件路径（只读，含 `live-state.json` / `decisions-active.json`）
  - 最后 checkpoint 时间

#### A5. Worker 池"加入 Team"补目标选择 `[P0]`
- "分配加入 Team" 按钮 → 改为下拉菜单或弹层
- 列出所有当前 Team（带 healthy/degraded 状态点）
- 选中后调用 claim 接口
- 若所有 Team 已满或不可加入，按钮置灰提示

### B. 核心信息补齐（P1）

#### B1. 左侧菜单 Teams/Worker 动态列表 `[P1]`
- 在固定菜单项下方加分组：
  - `━━ Teams ━━`
    - 🟢 Alpha (3) — 状态点 + Team 名 + 成员数
    - 🟡 Beta (1)
  - `━━ 工作池 ━━`
    - 🟢 idle (2)
    - 🔵 busy (1)
    - ⚫ offline (1)
- 单击进入对应内联视图，双击打开 Tab

#### B2. 决策中心筛选栏 `[P1]`
- 列表上方加筛选：
  - 紧急度（全部 / 紧急 / 常规）
  - Team（全部 / Alpha / Beta / ...）
  - （可选）状态（pending / resolved / expired）

#### B3. 删除"任务进度 12/15"伪字段 `[P1]`
- Team 卡片移除 "任务进度 12/15 ━━" 行
- 改为展示 `currentTask` 文本摘要（限 1 行，省略号截断）
- 同步：所有页面禁止再展示无数据模型支撑的进度数字

#### B4. 补关键异常状态 `[P1-部分]`
- **master offline** 状态：Team 卡片标题前显示 ⚫ 灰点 + "Master 离线 5min"
- **Team degraded** 状态：黄色边框 + 顶部"⚠️ Team 异常：worker-1 离线"提示
- **decision expired** 状态：决策项灰色 + "已过期 2h 前" + 操作按钮禁用
- **loading 占位**：网络请求时显示简单 loading 圆环（不做骨架屏精修）

#### B5. Worker 字段补齐 `[P1]`
- Team 详情成员列表 / Worker 池表格补齐：
  - `currentTaskSummary` — 当前任务摘要（busy 状态时展示）
  - `lastHeartbeat` — 最后心跳（相对时间，如"3s 前"）
  - `offlineSince` — 离线时长（offline 状态时展示）

#### B6. Decision 字段补齐 `[P1]`
- 决策列表项补齐：
  - `type` 标签（如 偏好选择 / 不可逆操作 / 需求歧义 / 高成本调用 / 任务超范围）
  - `expiresAt` 倒计时：
    - 为空 → 不展示
    - 距过期 < 10min → 红色高亮 "⚠️ 5min 后过期"
    - 已过期 → 灰色 "已过期 2h 前"

### C. 视觉收敛（P2）

#### C1. 按钮命名统一 `[P2]`
- "进入监控" / "进入完整监控" → 统一为 "打开监控"
- "成员快览" → 保留
- "分配加入 Team" → 统一为 "加入 Team"
- "解散 Team" → 保留

#### C2. 离线/紧急视觉权重提升 `[P2]`
- offline worker 行：背景灰色 + 状态点⚫ + 文字弱化
- 紧急决策项：左侧 4px 红色边条 + 紧急标签红底
- 顶部告警条：从细条改为略高（32px）

#### C3. Tab 状态色标统一 `[P2]`
- 所有 Team Tab 标题前必须有状态点：🟢 healthy / 🟡 degraded / ⚫ master offline
- 总览 Tab 用图标 📊（无状态点）

### D. 文档修订

#### D1. 修订需求文档矛盾 `[文档]`
- 文件：`docs/specs/2026-04-21-Agent-Team-需求文档.md`
- §5 "不做 web 面板"补充限定语：**仅适用于 MVP 范围**
