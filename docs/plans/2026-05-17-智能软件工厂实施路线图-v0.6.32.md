# 智能软件工厂实施路线图

> 版本：v0.6.32  
> 日期：2026-05-17  
> 状态：当前路线图 / 按多智能体协同与岗位责任重写

---

## 1. 路线图原则

智能软件工厂的研发过程本身也是智能软件工厂的半手工 POC。当前通过项目负责人手动调度需求设计岗、开发测试岗、交付验收岗协同推进；未来系统要自动完成这套调度。

阶段计划必须表达：

```text
目标
主责岗位
协作岗位
验收岗位
输入文档
输出产物
进度状态
```

---

## 2. 阶段总览

| 阶段 | 目标 | 主责岗位 | 协作岗位 | 验收岗位 |
|---|---|---|---|---|
| P0a | 前端可演示 MVP / Mock 闭环 | 需求设计岗 + 开发测试岗 | 交付验收岗 | 交付验收岗 + 项目负责人 |
| P0b | 本地可运行 MVP / 后端与持久化 | 开发测试岗 | 需求设计岗 | 交付验收岗 |
| P1 | 服务化与真实集成 | 开发测试岗 + 交付验收岗 | 需求设计岗 | 项目负责人 / 交付验收岗 |
| P2 | 多团队、多项目、多运行时增强 | Leader + 多岗位团队 | 多开发测试岗并行 | 交付验收岗 |

---

## 3. P0a：前端可演示 MVP

目标：无后端、无数据库、无真实 Gateway、无真实 OpenCode runtime，跑通可演示业务闭环。

岗位责任：

```text
需求设计岗：
  维护 PRD、系统设计、施工图、任务单、验收标准。

开发测试岗：
  建立 monorepo、contracts、domain、mock-seed、state、页面和自测。
  可以多个实例并行执行不同任务单。

交付验收岗：
  按任务单验收，不通过则提出整改任务单，通过则记录初验收报工。
```

P0a 交付：

```text
- pnpm workspace 工程骨架
- contracts/domain/mock-seed/data-client/state
- Skill 工作台
- AgentTemplate 管理
- Worker 创建/启用/停用模拟
- RuntimeHost / RuntimeBinding 展示
- Team / Project 页面
- 文档草稿/发布模拟
- 决策处理
- WorkOrder 任务单模型预留
- Activity 回写
- MockChatAdapter
```

P0a 闭环：

```text
Skill
→ AgentTemplate / 岗位
→ Worker + RuntimeBinding
→ Team / Project
→ Doc 发布
→ WorkOrder 派发
→ Decision 处理
→ 开发测试岗执行回执
→ 交付验收岗初验收
→ Activity 回写
```

## 5. P0b：本地可运行 MVP

目标：引入本地后端和持久化，但仍以单机验证为主。

可能交付：

```text
- Fastify API
- Prisma + SQLite
- Team/Worker/Skill/Doc/Decision/Activity/WorkOrder 持久化
- 本地 runtime-service
- OpenCode workspace 初始化
- Worker 启用/停用真实文件操作
- AgentRoute/AgentPresence 持久化
```

P0b 起，WorkOrder 可以从 Markdown 任务单升级为数据库对象。

---

## 6. P1：服务化和真实集成

目标：接入真实 agent-web-kit Gateway、OpenCode RuntimeHost、Provider 查询和生产级状态流。

交付：

```text
- PostgreSQL
- 认证/权限
- Gateway Provider API
- RuntimeHost heartbeat
- OpenCode supervisor
- ConversationRouteBinding 持久化
- WorkOrder 自动派发与状态流转
- Acceptance / Quality Gate 持久化
- 审计日志
```

---

## 7. P2：增强方向

```text
- 多项目模型
- 多团队资源调度
- 多个开发测试岗实例并行执行

- Skill marketplace
- 安全策略 RuntimePolicy
- 自动生成交接文档与复盘
```
