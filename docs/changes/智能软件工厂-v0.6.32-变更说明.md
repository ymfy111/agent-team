## 3. 核心变化

### 3.1 施工图细化

新增 P0a 工程目录结构、packages 职责、contracts 类型边界、domain 派生规则、mock-seed 场景、data-client 接口、state commands、页面落地顺序和验收标准。

### 3.2 Runtime 目录逻辑细化

补充 RuntimeHost 下 Worker workspace、skills 快照、binding、sync-status、conversation route binding 的未来目录结构。P0a 只在 mock 中展示，不真实写入。

### 3.3 OpenCode 执行清单

把 P0a 开发拆成 Group A-H：工程骨架、contracts、domain、mock-seed、data-client/state、Web App Shell、配置态页面、运行态闭环。

### 3.4 gstack 借鉴分析

考察 gstack 后，将可借鉴点转化为智能软件工厂设计建议：角色化 Skill、计划审查、可执行任务、安全护栏、worktree 并行、多模型二次审查、文档同步、上下文保存与恢复。

### 3.5 计划评审模板

新增 OpenCode 开工前评审模板，要求在复杂任务前先说明模块边界、数据流、状态流转、信任边界和验收方式。

## 4. 不变内容

```text
- P0a 仍不接真实后端。
```
