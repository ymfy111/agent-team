```text

```

如果用户说“还没同步”，则应基于上一轮会话输出的 zip / 文件继续，不要假设 GitHub 已包含最新内容。

### 1.3 文件生成规则

后续继续生成文件时：


## 4. 核心产品与架构决策

### 4.1 岗位、员工、技能

当前概念冻结：

```text
岗位 = AgentTemplate / 数字员工模板
员工 = Worker / AgentInstance / 数字员工实例
技能 = Skill / SKILL.md 能力资产
岗位技能匹配 = TemplateSkillMapping
项目匹配 = ProjectAssignment
```

规则：

```text
- Skill 是能力资产。
- SkillVersion 是技能内容真源。
- TemplateSkillMapping 是岗位技能匹配真源。
- Worker.skills 是派生视图，不是主数据。
- Worker 从 AgentTemplate 创建，并继承模板匹配的 Skill。
```

### 4.2 数字员工运行时模型

当前设计核心：

```text
一个数字员工实例 Worker
= 一个具体 Agent Runtime
= 当前实现形态为某个 RuntimeHost 上的 OpenCode 工作目录
```

补充：

```text
- OpenCode Runtime 可以单机部署，也可以多 RuntimeHost 部署。
- 不能假设所有 Worker workspace 都在同一台服务器。
- WorkerRuntimeBinding 绑定 workerId、runtimeHostId、workspaceDir、skillDir、初始化技能快照和 syncStatus。
- workspaceDir 中的 Skill 文件只是运行时快照，不是技能主数据。
```

### 4.3 员工启用 / 停用

启用动作：

```text
1. 选择或自动分配 RuntimeHost
2. 创建或复用 OpenCode workspaceDir
3. 根据 AgentTemplate 和 TemplateSkillMapping 查 SkillVersion
4. 初始化 SKILL.md / scripts / resources 到 workspaceDir
5. 启动或绑定 OpenCode runtime
6. 生成或激活 AgentRoute
7. 标记 lifecycleStatus = enabled
8. 标记 runtimeStatus = running
9. 生成 Activity
```

停用动作：

```text
1. 停止对应 OpenCode runtime
2. workspaceDir 不删除
3. skill 初始化快照不删除
4. AgentRoute.routeStatus = disabled
5. Worker.lifecycleStatus = disabled
6. runtimeStatus = disabled
7. 只允许重新启用
```

停用后禁止：

```text
- 重启
- 下线
- 同步技能
- 对话
```

除非重新启用。

## 5. AgentRoute / AgentPresence / Gateway 设计

### 5.1 前端 agentId-only 原则

前端 / widget 只传：

```text
agentId
conversationId
hostContext
```

不传：

```text
endpointRef
workspaceDir
runtimeHost
OpenCode port
token
```

### 5.2 AgentRoute

当前统一字段：

```ts
type AgentRoute = {
  agentId: string
  ownerType: 'global' | 'leader' | 'worker'
  ownerId: string
  routeRevision: number
  routeStatus: 'active' | 'disabled' | 'unavailable' | 'error'
  runtimeType: 'opencode' | 'mock' | 'custom'
  runtimeHostId?: string
  endpointRef: string
  workspaceDir?: string
  capabilities?: string[]
}
```

注意：不要再新增 `AgentRoute.status`。如历史适配层接收到 `status`，进入 domain / selector / DTO 前必须归一化为 `routeStatus`。

### 5.3 AgentPresence

```ts
type AgentPresence = {
  agentId: string
  runtimeHostId?: string
  routeStatus: 'active' | 'disabled' | 'unavailable' | 'error'
  presenceStatus: 'online' | 'offline' | 'busy' | 'idle' | 'starting' | 'stopping' | 'failed'
  currentConversationId?: string
  currentTaskSummary?: string
  lastHeartbeatAt?: string
  updatedAt: string
}
```

### 5.4 routeRevision

递增规则：

```text
必须递增：
- endpointRef 变化
- runtimeHostId 变化
- workspaceDir 变化
- runtimeType 变化
- routeStatus 从 active 变为 disabled / unavailable / error
- capabilities 发生影响会话能力的变化

不应单独递增：
- lastHeartbeatAt 更新
- presenceStatus 更新
- currentTaskSummary 更新
- metrics 更新
```

### 5.5 ConversationRouteBinding

conversation 创建时保存 route snapshot：

```ts
type ConversationRouteBinding = {
  conversationId: string
  agentId: string
  routeRevision: number
  runtimeType: 'opencode' | 'mock' | 'custom'
  runtimeHostId?: string
  endpointRef: string
  workspaceDir?: string
  boundAt: string
}
```

规则：

```text
- 新 conversation 使用最新 AgentRoute。
- 已有 conversation 默认继续使用创建时的 ConversationRouteBinding。
- routeStatus = disabled 后拒绝新 conversation。
- 已有 conversation 后续 sendCommand 可返回 AGENT_DISABLED / AGENT_UNAVAILABLE，或提示重新创建会话。
- ConversationRouteBinding 是后端 / Gateway 内部数据，不暴露给浏览器前端。
```

### 5.6 心跳到底走不走 Gateway

当前定案：

```text
正式业务系统集成：
RuntimeHost / OpenCode supervisor
→ agent-team runtime-service
→ agent-team 数据库保存 Worker / RuntimeHost / AgentRoute / AgentPresence
→ agent-gateway 通过 Provider 查询路由和脱敏状态

轻量 / Mock / Demo 集成：
RuntimeHost / Mock runtime
→ agent-gateway heartbeat
→ Gateway 自己维护 route + presence 临时状态
```

也就是说：

```text
Gateway 支持心跳；
但智能软件工厂正式集成时，Worker 在线 / 离线 / 繁忙 / 空闲状态的业务真源是 agent-team，不是 Gateway。
```

### 5.7 会话级 busy 与员工级 busy

```text
会话级 busy：
当前 conversation 正在 streaming、工具调用或等待 interaction。
可由 Gateway / widget-core 即时感知。

员工级 busy：
某个业务智能体当前正在执行任务、处理项目动作或不可接新任务。
正式业务系统中应回写到 agent-team 的 AgentPresence / Worker 状态。
```

Gateway 可以产生会话级 busy，但不应自动等同为员工级 busy。

### 5.8 RuntimeHost offline 推导

```text
- RuntimeHost offline：其下 active route 可视为 unavailable。
- RuntimeHost offline：其下 presenceStatus 可视为 offline。
- 不覆盖 disabled route；disabled 仍保持 disabled。
- RuntimeHost 恢复 online 后，不应批量自动把所有 Agent 改为 active。
- Host 恢复后，每个 Agent 仍需等待 Agent heartbeat 或 Provider 状态恢复。
```
