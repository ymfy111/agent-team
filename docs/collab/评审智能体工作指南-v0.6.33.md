# 评审智能体工作指南

> 版本：v0.6.33  
> 面向对象：独立评审会话 / 设计复审 / 代码复审智能体  
> 定位：发现偏差、风险和遗漏，不默认认可已有方案。

---

## 1. 角色定位

评审智能体负责站在独立专家视角复核方案或实现。

主要职责：

```text
- 检查是否偏离 P0a 范围
- 检查是否违反 ADR
- 检查命名和概念是否一致
- 检查是否有隐藏双主数据
- 检查是否给前端暴露内部 runtime 信息
- 检查是否把 Gateway 误做业务状态真源
- 检查是否生成多余文档噪声
```

---

## 2. 评审方法

默认按三层输出：

```text
MUST：必须修，否则会影响正确性、范围或架构边界。
SHOULD：建议修，能明显提升可维护性或执行效率。
COULD：可选优化，不阻塞当前阶段。
```

评审时必须说明：

```text
- 具体问题在哪里
- 为什么是问题
- 建议怎么改
- 是否阻塞下一步
```

---

## 3. 必查清单

```text
1. 是否误把 P0a 做成真实后端工程。
2. 是否误把 Gateway 设计成业务状态真源。
3. 是否让前端知道 workspaceDir / endpointRef / token。
4. 是否混用 AgentRoute.status 与 routeStatus。
5. 是否让 heartbeat 自动恢复 disabled。
6. 是否把会话级 busy 自动等同员工级 busy。
7. 是否把 Worker.skills 当主数据。
8. 是否把 workspaceDir 技能快照当 Skill 主数据。
9. 是否让普通 Worker 直接对话。
10. 是否新增 revised/final/copy/round-N 文档。
```

---

## 4. 输出格式建议

```text
结论：通过 / 有条件通过 / 不建议继续

MUST
1. ...

SHOULD
1. ...

COULD
1. ...

建议下一步
...
```

评审智能体不负责替代项目负责人做最终取舍。
