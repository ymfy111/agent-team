# 智能体协作分工

> 版本：v0.6.32  
> 状态：当前半手工 POC 的协作规则 / 未来智能软件工厂调度模型参考

---

## 1. 当前项目为什么需要协作分工

智能软件工厂系统尚未实现前，外部智能体无法自动知道：

```text
我是什么岗位？
我负责哪个项目？
我应该读哪些文档？
当前阶段是什么？
本轮任务单是什么？
遇到阻塞和分歧应该找谁？
什么情况需要向用户上报？
做完后如何验收和回写？
```

因此，当前由项目负责人手动调度多个外部智能体协作。这个半手工流程本身就是智能软件工厂未来产品化的 POC。

---

## 2. AI 原生岗位，而不是人类岗位搬运

智能软件工厂中的岗位不是人类岗位的简单映射，而是面向大模型工作方式设计的 AI 原生岗位。

人类软件工程会拆成产品、需求、架构、前端、后端、测试、QA、项目经理等岗位，是因为人类能力和时间有限。大模型智能体具备更强跨职能能力，因此更适合少量复合岗位完成端到端任务，再在关键节点通过独立审查和质量门禁控制风险。

当前核心岗位：

```text
协同规划岗
实现验证岗
交付审查岗
```

---

## 3. 岗位与团队角色

### 3.1 协同规划岗

默认担任团队 Leader / 组长。

职责：

```text
- 和用户沟通需求。
- 把用户意图沉淀到 PRD、设计、计划和任务单。
- 维护阶段目标、任务优先级和任务依赖。
- 跟踪任务进度、阻塞、风险和分歧。
- 对普通分歧先做初判。
- 超出判断范围时生成 Decision / ChangeRequest 上报用户。
- 收集实现验证岗的执行回执和交付审查岗的验收结论。
- 向用户反馈进度、阻塞、决策点和交付结果。
```

说明：

```text
Leader / 组长 是团队角色，不是单独岗位。
默认由协同规划岗数字员工担任。
```

### 3.2 实现验证岗

职责：

```text
- 按任务单编码。
- 使用 TDD / 单元测试 / 局部集成测试完成自测。
- 可调用 OpenCode 内部子智能体做局部代码审查、测试补充、风险扫描。
- 修复自测问题。
- 输出执行回执。
```

说明：

```text
实现验证岗的自测不等于交付放行。
实现验证岗可以有多个数字员工实例并行。
每个实例可对应一个 OpenCode 独立进程 / workspace / 主智能体。
```

### 3.3 交付审查岗

职责：

```text
- 读取需求、设计、任务单、执行回执和验收标准。
- 使用独立视角做质量门禁。
- 可使用独立会话、独立模型或独立浏览器 session。
- 批量输出问题清单、风险等级、整改建议。
- 不通过时生成整改任务单。
- 通过时记录初验收结果和验收报工。
```

说明：

```text
交付审查岗不参与实现验证岗的日常 TDD 循环。
验收不做碎片化即时往返，而采用批量门禁。
```

## 4. 当前半手工 POC 映射

```text
项目负责人
  最终决策者、人工调度者。

设计智能体 / ChatGPT
  当前主要承担协同规划岗。
  必要时切换为技术架构评审模式或交付审查视角，但必须声明视角切换。

编程智能体 / OpenCode / Codex
  承担实现验证岗。
  每个独立 OpenCode 进程可视为一个数字员工运行体。

独立会话 / 不同模型 / 独立 OpenCode 进程
  可承担交付审查岗。
```

## 5. 阻塞、分歧和对人反馈链路

```text
一级：岗位内自处理
  实现验证岗遇到普通代码错误、测试失败、局部实现问题，先自己 TDD / 自测 / 修复。

二级：Leader 协调
  遇到需求不清、任务范围冲突、实现路径分歧、依赖阻塞，提交给协同规划岗 / Leader 初判。

三级：人类决策
  Leader 仍无法裁决，或涉及需求取舍、优先级变化、不可逆操作、交付范围变化，则生成 Decision / ChangeRequest，上报用户处理。
```

---

## 6. 任务执行链路

```text
确认岗位 / 团队角色
→ 读取工作指南
→ 读取项目上下文
→ 对齐当前阶段和进度
→ 接收 WorkOrder / 任务单
→ 执行任务
→ 输出执行回执
→ 进入评审 / 验收 / 决策 / 下一轮任务
```

---

## 7. 文档交接原则

智能体之间默认通过文档和任务单交接，不采用高频闲聊式协作。

```text
协同规划岗 → 实现验证岗：通过任务单、输入文档、范围和验收标准交接。
实现验证岗 → 协同规划岗：通过执行回执、阻塞、风险和产物说明回写。
实现验证岗 → 交付审查岗：通过执行回执、测试结果、截图、构建结果交接。
交付审查岗 → 协同规划岗 / 实现验证岗：通过验收结论、整改清单、整改任务单回写。
```

## 8. 和未来智能软件工厂的关系

当前半手工协作方式在未来产品中应转化为：

```text
AgentTemplate / 岗位模板
Worker / 数字员工实例
TeamRole / Leader 与成员
WorkOrder / 任务单
Progress / 进度
Blocker / 阻塞
Decision / 决策
ChangeRequest / 变更
AcceptanceRecord / 验收记录
Activity / 活动回写
```
