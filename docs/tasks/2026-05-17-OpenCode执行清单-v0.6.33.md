# OpenCode 执行清单

> 版本：v0.6.33  
> 日期：2026-05-17  
> 状态：施工执行清单 / 不直接编码版  
> 用途：给 OpenCode / Codex 在后续真正开发时使用。  
> 前提：本清单不要求当前 ChatGPT 会话写代码；ChatGPT 负责设计和任务说明，OpenCode 负责按任务创建文件、编码、测试和提交。

---

## 1. 总体执行原则

OpenCode 必须按以下顺序执行：

```text
文档读取
→ 工程骨架
→ contracts
→ domain
→ mock-seed
→ data-client
→ state
→ Web App Shell
→ 配置态页面
→ 运行态页面
→ P0a E2E
```

禁止跳过 contracts/domain/mock-seed 直接做页面。

## 3. 任务分组

### Group A：工程骨架

#### TASK-A01 初始化 workspace

目标：创建 pnpm workspace 和基础 TypeScript 配置。

允许创建：

```text
package.json
pnpm-workspace.yaml
tsconfig.base.json
.gitignore
apps/web/package.json
apps/web/tsconfig.json
packages/*/package.json
packages/*/tsconfig.json
```

禁止：

```text
不创建 apps/api。
不引入 Prisma。
不引入数据库。
不引入真实 agent-web-kit 依赖。
```

验收：

```bash
pnpm install
pnpm -r typecheck
```

提交建议：

```text
chore: initialize p0a workspace skeleton
```

---

### Group B：contracts

#### TASK-B01 定义核心 ID 和通用类型

涉及目录：

```text
packages/contracts/src/ids.ts
packages/contracts/src/common.ts
packages/contracts/src/index.ts
```

验收：所有后续类型引用统一 ID 类型，不在业务文件重新声明 string alias。

#### TASK-B02 定义 Skill / SkillVersion / AgentTemplate / TemplateSkillMapping

涉及目录：

```text
packages/contracts/src/skill.ts
packages/contracts/src/agent-template.ts
```

验收：能够表达通用技能、模板技能、技能版本和影响范围。

#### TASK-B03 定义 Worker / RuntimeHost / WorkerRuntimeBinding

涉及目录：

```text
packages/contracts/src/worker.ts
packages/contracts/src/runtime.ts
```

验收：Worker 不包含 skills 主字段；RuntimeBinding 包含 workspaceDir、skillDir、initializedSkillSnapshot、syncStatus。

#### TASK-B04 定义 AgentRoute / AgentPresence / ConversationRouteBinding

涉及目录：

```text
packages/contracts/src/route-presence.ts
packages/contracts/src/chat.ts
```

验收：AgentRoute 只有 routeStatus，没有 status；ConversationRouteBinding 不暴露给页面 UI。

#### TASK-B05 定义 Team / Project / Doc / Decision / Activity

涉及目录：

```text
packages/contracts/src/team-project.ts
packages/contracts/src/document.ts
packages/contracts/src/decision.ts
packages/contracts/src/activity.ts
```

验收：Doc 支持草稿与发布版本；Decision 支持 pending/resolved/blocked/expired；Activity 可表达 Skill、Doc、Decision、Worker 事件。

提交建议：

```text
feat(contracts): add p0a domain contracts
```

### Group C：domain

#### TASK-C01 实现 deriveWorkerSkills

输入：Worker、AgentTemplate、Skill、SkillVersion、TemplateSkillMapping。  
输出：DerivedWorkerSkill[]。

必须覆盖：

```text
通用技能
模板技能
disabled Skill 过滤
currentVersionId 解析
source 标记
order 排序
```

#### TASK-C02 实现 computeSkillImpact

必须输出：

```text
affectedTemplateIds
affectedWorkerIds
affectedRuntimeBindingIds
needRuntimeSyncWorkerIds
```

#### TASK-C03 实现 runtime syncStatus 计算

必须覆盖：

```text
synced
pending
out_of_sync
failed
not_initialized
host offline 不直接改 failed
worker disabled 禁止同步动作
```

#### TASK-C04 实现 route / presence 有效状态推导

必须覆盖：

```text
RuntimeHost offline → active route 视为 unavailable
disabled route 不被覆盖
host online 不自动恢复 disabled
会话级 busy 不等于员工级 busy
```

#### TASK-C05 实现 canWorkerChat

必须覆盖：

```text
Leader 可直接对话
普通 Worker 通过 Leader
未入组 Worker 不可对话
route disabled/unavailable/error 不可新建会话
```

提交建议：

```text
feat(domain): add p0a derived rules and invariants
```

## 4. 回归检查清单

每轮实现后检查：

```text
[ ] 没有新增 AgentRoute.status。
[ ] 页面没有直接写 mock 常量数组。
[ ] Worker 类型没有主数据 skills 字段。
[ ] disabled route 不会被 host online 恢复。
[ ] RuntimeHost offline 推导只影响 effective view，不覆盖原 routeStatus。
[ ] 普通 Worker 不创建直接会话。
[ ] Doc 发布和 Decision 处理都写 Activity。
[ ] P0a 未引入后端/数据库/Gateway/OpenCode 真实集成。
```

## 5. 失败时处理

如果 OpenCode 遇到设计不清楚：

```text
1. 不要自行扩大范围。
2. 在 docs/reports/ 下新增一个“实现阻塞记录”。
3. 写清楚冲突文档、影响模块、推荐方案。
4. 停在当前任务边界，不继续往后猜。
```
