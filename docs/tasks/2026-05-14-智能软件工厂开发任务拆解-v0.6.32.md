## 1. 任务编写模板

每个开发 issue 必须使用统一模板，避免执行时自由发挥。

```text
任务 ID：
任务名称：
阶段：P0a / P0b / P1
层级：第一层基础骨架 / 第二层最小闭环 / 第三层页面补全 / P0b / P1
关联需求目标：对应开发计划中的需求目标
关联设计模块：对应系统设计中的模块 / 数据模型 / 适配层
关联验收方式：对应测试用例、P0a DoD 或 E2E 故事
目标：本任务要完成什么业务或技术结果
输入：依赖哪些文档、模型、mock 数据、上游任务
输出：要新增或修改哪些页面、模块、包、接口或数据
涉及文件 / 模块：apps/...、packages/...、docs/...
前置依赖：必须先完成的任务 ID
验收条件：可观察、可点击、可测试的完成标准
不做范围：本任务明确不处理的内容
截图要求：如涉及页面，说明需要提供哪些截图
```

验收条件必须写成可判断结果，避免只写“支持、完成、优化”。

## 2. P0a 三层执行顺序

P0a 不按页面完整度推进，而按“最小闭环主线”推进。

```text
第一层：不可跳过的基础骨架任务
→ 第二层：最小闭环任务
→ 第三层：页面补全与体验增强任务
```

强约束：

```text
第一层未完成前，不进入页面功能开发。
第二层未跑通前，不做视觉增强、复杂筛选、批量操作或真实集成。
```

P0a 唯一主线：

```text
Skill → AgentTemplate → Worker + RuntimeBinding → Team/Project → Doc 发布 → Decision 处理 → Activity 回写
```

## 9. Issue 创建建议

- 每个任务单独创建 issue。
- 每个 issue 必须使用本文第 1 节任务模板，至少包含：关联需求目标、关联设计模块、关联验收方式、目标、输入、输出、涉及文件 / 模块、前置依赖、验收条件、不做范围、截图要求。
- P0a 不阻塞于后端；所有写操作先用 mock/local state 模拟。
- 涉及 `Skill`、`AgentTemplate`、`Worker` 的任务必须遵守：Skill 是资产，AgentTemplate 是模板，Worker 是实例。
- 涉及数字员工运行时的任务必须遵守：RuntimeHost 是运行节点，WorkerRuntimeBinding 是 Worker 到 OpenCode 工作目录的绑定，workspaceDir 内技能只是运行时快照。
- 涉及小云的任务必须遵守：普通员工不直接对话，必须通过所属团队 Leader。

### RUNTIME-003：建立 AgentRoute 与 AgentPresence mock 模型

阶段：P0a  
层级：第一层基础骨架  
关联需求目标：数字员工运行时管理、小云连接能力  
关联设计模块：AgentRoute、AgentPresence、AgentRouteProvider、RuntimeHost  
关联验收方式：员工可从 mock seed 解析 agentId、routeStatus、presenceStatus；disabled 员工只能重新启用。  

目标：在 contracts、mock-seed、state/selectors 中补齐 AgentRoute 与 AgentPresence。  
输入：系统设计 §6.10、Mock 数据设计 §8。  
输出：agentRoutes[]、agentPresences[]、selectors。  
涉及文件 / 模块：packages/contracts、packages/mock-seed、packages/state。  
前置依赖：RUNTIME-001。  
验收条件：员工卡可展示启用状态、路由状态、在线/离线/忙闲；Gateway 地址不暴露给前端。  
不做范围：不接真实 agent-gateway，不写真实心跳接口。  

### RUNTIME-004：员工启用 / 停用 mock 流程

阶段：P0a  
层级：第二层最小闭环  
关联需求目标：数字员工管理、运行时工作区模型  
关联设计模块：Worker.lifecycleStatus、WorkerRuntimeBinding、AgentRoute、AgentPresence  
关联验收方式：启用后 routeStatus=active、presenceStatus=idle；停用后 routeStatus=disabled、presenceStatus=offline。  

目标：实现员工启用 / 停用的前端 mock 状态流转。  
输入：Worker、WorkerRuntimeBinding、AgentRoute、AgentPresence mock。  
输出：启用 / 停用按钮状态、Activity 回写、状态刷新。  
涉及文件 / 模块：apps/web 员工页、packages/domain、packages/state。  
前置依赖：RUNTIME-003、WORKER-001。
