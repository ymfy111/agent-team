## 4. 禁止事项

```text
1. 不实现真实后端 API。
2. 不接真实数据库。
3. 不接真实 Gateway。
4. 不启动真实 OpenCode Runtime。
5. 不接真实 agent-browser。
6. 不做完整页面。
7. 不恢复传统架构/开发/测试/设计/建模岗位模型。
8. 不新增 AgentRoute.status，只使用 routeStatus。
9. 不在浏览器前端暴露 endpointRef / workspaceDir / token。
10. 不修改 docs 正本文档结构。
```

要求：

```text

- Leader 是 teamRole，不是单独岗位模板。
- WorkOrder 支持 reviewStrategy / acceptanceStrategy / browserSessionPolicy。
- AcceptanceRecord 支持 findings / evidenceRefs / reworkWorkOrderIds。
```

## 6. 必须实现的 domain selector / rule

至少实现：

```text
selectTeamLeader(teamId)
selectTeamMembers(teamId)
selectWorkersByTemplate(templateId)
selectWorkOrdersByTeam(teamId)
selectActiveWorkOrdersByWorker(workerId)
selectTeamProgressSummary(teamId)
selectBlockingItems(teamId)
selectPendingDecisions(teamId)
selectPendingAcceptances(teamId)
selectActivitiesByWorkOrder(workOrderId)
```

规则：

```text
1. Leader 从 team.leaderWorkerId 推导。
2. Worker.skills 从 Skill / SkillVersion / TemplateSkillMapping 派生。
3. 实现验证岗的自测状态不等同于交付通过。
4. independent_acceptance 策略下，验收 Worker 不应等于执行 Worker。
5. RuntimeHost offline 只推导 unavailable/offline，不自动恢复 disabled。
```

## 8. 验收标准

交付审查岗会按以下标准初验：

```text
1. workspace 基础文件存在。
2. contracts 包可以独立导出核心类型。
3. domain 包可以引用 contracts 类型。
4. AI 原生岗位模型存在，未退回传统岗位模型。
5. WorkOrder / Progress / AcceptanceRecord / BrowserRuntimeSession 类型存在。
6. routeStatus 未被 status 混用。
7. 前端敏感运行时字段没有暴露给页面组件。
8. P0a 禁止项未被实现。
```
