# 编程智能体执行清单

> 版本：v0.6.32  
> 状态：给 OpenCode / Codex 的阶段执行顺序

---

## 1. 角色确认

编程智能体在当前项目中承担：

```text
实现验证岗
```

职责：

```text
按任务单编码、TDD、自测、局部集成、修复并输出执行回执。
```

编程智能体可以调用运行时内部子智能体做局部代码审查、测试补充和风险扫描，但这属于自检能力，不等于交付审查通过。

## 3. 阶段执行顺序

```text
第一轮：workspace + contracts + domain
第二轮：mock-seed
第三轮：data-client + state
第四轮：apps/web app shell
第五轮：总览 / 团队工作台 / 员工 / 决策页面
第六轮：交付审查视图 + 浏览器自测/验收占位
第七轮：视觉和交互打磨
```

当前只执行第一轮，除非项目负责人明确下发下一张任务单。

---

## 6. 遇到问题的处理

```text
普通代码问题：自行修复。
类型边界不清：先按当前正本文档实现，并在回执中标注。
任务范围冲突：停止扩大范围，提交给协同规划岗。
需求/设计冲突：不要自行改需求，提交 Decision / ChangeRequest 建议。
```
