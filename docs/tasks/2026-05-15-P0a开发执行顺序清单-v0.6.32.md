# P0a 开发执行顺序清单

> 版本：v0.6.32  
> 日期：2026-05-15  
> 状态：P0a 开工准备稿  
> 依据：开发计划、开发任务拆解、Mock 数据设计、P0a 闭环演示原型。

## 1. 使用原则

本清单用于约束 OpenCode / Codex / 多智能体开发顺序。开发时优先按本清单推进，不按页面视觉完整度随意开工。

P0a 主线：

```text
Skill → AgentTemplate → Worker + RuntimeBinding → Team/Project → Doc 发布 → Decision 处理 → Activity 回写
```

## 5. 禁止提前做

P0a 阶段不要提前做：

```text
- 真实后端 API
- SQLite / Prisma
- 真实 agent-web-kit / Gateway / OpenCode
- 真实创建或写入 OpenCode 工作目录
- 多 RuntimeHost 调度
- 登录、权限、多用户、完整审计
- 复杂批量操作和生产级性能优化
```

## 6. P0b 进入条件

只有满足以下条件后，才进入 P0b：

1. P0a E2E 主线跑通。
2. mock seed 和 selectors 稳定。
3. RuntimeHost / WorkerRuntimeBinding 的 mock 结构被页面使用。
4. Skill 发布、模板挂载、Worker 创建、文档发布、决策处理、Activity 回写均可在前端验证。

## 补充：Route / Presence 开发边界

P0a / P0b / P1 的边界如下：

```text
P0a：
- 只做 mock seed / mock selectors / mock 状态展示。
- routeStatus、presenceStatus、routeRevision、ConversationRouteBinding 只作为前端 mock 数据。
- 不接真实 Gateway，不做真实 heartbeat。

P0b：
- 做 agent-team API 与本地 runtime-service 的最小实现。
- 支持员工启用 / 停用、本地 OpenCode workspace 初始化、AgentRoute / AgentPresence 持久化。
- Gateway Provider 可先不真实接入。

P1：
- 接入真实 agent-gateway Provider。
- RuntimeHost / OpenCode supervisor 向 agent-team runtime-service 上报心跳。
- Gateway 通过 Provider 查询脱敏 route / presence。
```

开发顺序补充：

```text
01. contracts 增加 routeStatus / presenceStatus / routeRevision
02. mock-seed 增加 AgentRoute / AgentPresence / ConversationRouteBinding
03. selectors 实现 RuntimeHost offline 推导
04. Worker 页面展示启用 / 停用 / routeStatus / presenceStatus
05. P0a E2E 覆盖 disabled 不可对话、重新启用恢复 mock 状态
```
