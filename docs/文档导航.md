## 目录使用规则

- `docs/specs/`：正式需求、系统设计、专项集成方案、P0a 施工图、Mock 数据设计。
- `docs/plans/`：阶段计划、路线图、开发计划。
- `docs/tasks/`：编程智能体可执行任务清单、任务单。
- `docs/collab/`：设计智能体、编程智能体、评审智能体的分工与工作指南。
- `docs/decisions/`：ADR / 架构决策 / 产品关键决策。
- `docs/changes/`：版本变更说明。
- `docs/reports/`：评审报告、走查报告、测试报告。
- `docs/prototypes/`：HTML 原型与视觉资源。
- `docs/references/`：外部项目参考资料；采纳后需转化为 specs 或 ADR。
- `docs/guides/`：协作方法、文档规范、操作指南。

## 维护约定

1. 新增文档必须归类到以上目录，并在本文件中补充对应入口。
2. 阶段状态发生变化时，及时更新阶段总览表格。
3. 原型文件版本化保存，不覆盖历史版本。
4. 项目事实写入 `docs/project-memory.md`；智能体运行状态写入本地智能体记忆目录。
5. 生成文件包应保持与仓库目录一致，便于本地解压覆盖。
6. 当前最新稳定版本保持 v0.6.32；小改不递增版本号，只有里程碑级变化才升版本。
