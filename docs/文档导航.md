## 目录使用规则

- `docs/specs/`：放当前项目正式规格和设计方案，包括 PRD、系统设计、专项集成方案。
- `docs/plans/`：放阶段计划、Roadmap、开发计划。
- `docs/tasks/`：放 issue 级开发任务拆解。
- `docs/decisions/`：放 ADR / 架构决策 / 产品关键决策，不放变更说明。
- `docs/changes/`：放版本变更说明。
- `docs/reports/`：放评审报告、走查报告、测试报告。
- `docs/references/`：放外部项目参考资料；采纳后需转化为 specs 或 ADR。
- `docs/guides/`：放协作方法和文档规范。

## 维护约定

1. 新增文档必须归类到以上目录，并在本文件中补充对应入口。
2. 阶段状态发生变化时，及时更新阶段总览表格。
3. 原型文件版本化保存，不覆盖历史版本。
4. 项目事实写入 `docs/project-memory.md`；智能体运行状态写入本地智能体记忆目录。
5. GitHub 仓库作为后续文档源；生成的文件包应保持与仓库目录一致，便于本地解压覆盖。
6. 当前工作版本保持 v0.6.32；小改不递增版本号，只有里程碑级变化才升版本。
