## 5. ADR / 架构决策

| ADR | 文档 | 结论 |
|---|---|---|
| ADR-0001 | `docs/decisions/ADR-0001-agent-web-kit非侵入式集成.md` | agent-team 不复制聊天 UI，只通过 agentId + hostContext 接入。 |
| ADR-0002 | `docs/decisions/ADR-0002-agent-team采用monorepo分层架构.md` | P0a 使用 apps/web + packages 分层。 |
| ADR-0003 | `docs/decisions/ADR-0003-agent-team作为业务状态真源.md` | 正式集成中 Worker / RuntimeHost / Route / Presence 真源在 agent-team。 |
| ADR-0004 | `docs/decisions/ADR-0004-P0a仅做Mock前端闭环.md` | P0a 不接真实后端、真实 Gateway、真实 OpenCode。 |
| ADR-0005 | `docs/decisions/ADR-0005-Worker与RuntimeBinding模型.md` | 一个 Worker 绑定一个具体 Agent Runtime，当前表现为 RuntimeHost 上的 workspace。 |
| ADR-0006 | `docs/decisions/ADR-0006-前端agentId-only原则.md` | 前端只传 agentId / conversationId / hostContext，不暴露 endpointRef / workspaceDir / token。 |

## 8. 文档正本规则

```text
1. 不再维护 docs/README.md。
2. 不再维护子目录 README.md。
3. docs/文档导航.md 是唯一详细索引。
4. 同一主题只保留一个当前正本。
5. 历史版本、过程稿、revised/final/copy/round-N 文件不进入当前 docs。
6. 重大设计决策写 ADR。
7. 多轮评审结果收敛成一份 reports 当前报告。
8. 旧文档靠 Git 历史追溯，不在当前 docs 中复制保留。
```

## 9. 后续工作方式

本次 v0.6.33 是一次性过期文档清理，适合全量替换 `docs/`。本次之后恢复日常小步迭代：

```text
- 当前任务改到哪些文档，就只更新哪些文档。
- 不再每次都生成完整 docs 包。
- 不再新增多份过程稿。
- 小改用 Git commit 追踪。
```
