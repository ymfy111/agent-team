## 命名规范

- 中文撰写、中文语义命名。
- 日期前缀：`YYYY-MM-DD-文档标题.md`。
- 决策记录：`ADR-NNNN-简短中文标题.md`。
- 原型文件需版本化，不覆盖历史版本。
- 集成方案属于专项设计规格，放入 `docs/specs/`。
- 变更说明放入 `docs/changes/`。
- 架构/产品关键取舍放入 `docs/decisions/`。

## 维护约定

1. 新增文档必须归类到以上目录，并在「当前文档全览」树中补充对应行。
2. 阶段状态发生变化时，及时更新阶段总览表格。
3. 原型文件版本化保存，不覆盖历史版本。
4. 项目事实写入 `docs/project-memory.md`；智能体运行状态写入 p1 的 `.opencode/mem/`。
5. GitHub 仓库作为后续文档源；ChatGPT 生成的文件包应保持与仓库目录一致，用户本地解压/提交。
