# 决策记录（ADR）

本目录用于记录项目中的重要架构决策、产品边界决策和技术路线决策。

## 使用规则

- 记录“为什么这样选”，不只记录“选了什么”。
- 一个 ADR 只描述一个重要决策。
- 普通版本变更说明不要放在本目录，应放入 `docs/changes/`。
- 已接受的 ADR 原则上不直接改结论，如需改变，新增后续 ADR 覆盖。
