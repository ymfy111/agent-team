# 智能软件工厂产品需求规格

> 版本：v0.6.33  
> 日期：2026-05-17  
> 状态：当前正本 / 由 v0.6.32 多轮需求稿合并  
> 范围：产品能力、页面、业务对象、交互闭环、验收条件。技术实现细节见系统设计和 P0a 工程施工图。

---

## 1. 产品定位

智能软件工厂是一个智能体协作系统。它把多个 AI 智能体组织成有岗位、有技能、有团队、有项目、有文档和有决策闭环的数字员工团队。

用户不是直接面对一个通用聊天框，而是管理一组数字员工：

```text
团队 Leader 负责沟通与协调
普通成员负责项目中的专业工作
文档提供执行上下文
决策中心处理人在环确认
活动流形成可追踪执行记录
```

---

## 2. 目标用户

1. 需要组建 AI 研发团队的项目负责人。
2. 需要查看智能体状态、项目进度和阻塞项的技术 owner。
3. 希望用需求、计划、设计、决策和报告驱动 AI 团队开发的人。
4. 希望未来让 OpenCode / Codex 等 agent runtime 接入统一团队管理界面的人。

---

## 3. 核心价值

| 价值 | 说明 |
|---|---|
| 团队化 | AI 不再是零散工具，而是可识别、可分配、可监控的数字员工团队。 |
| 岗位化 | 通过 AgentTemplate 定义岗位，通过 Worker 实例化多个同岗位员工。 |
| 技能化 | Skill / SkillVersion 是能力资产，可复用、可匹配岗位、可初始化到运行时。 |
| 项目化 | 每个团队承接项目，项目包含文档、仓库、阶段、阻塞与动态。 |
| 文档驱动 | 文档发布才代表把上下文交给 Agent 执行或参考。 |
| 决策闭环 | Agent 遇到歧义、风险、偏好或不可逆动作时上报决策，由人处理后回写。 |
| 可观察 | 团队状态、员工状态、运行时状态、路由状态、活动流可统一查看。 |


---

## 3.1 从人工调度智能体到智能软件工厂调度

当前智能软件工厂系统尚未实现前，多个外部智能体无法自动获得统一的岗位、项目、任务、文档和决策上下文。实际协作仍依赖项目负责人手动调度：先与设计智能体沟通形成需求、架构、施工图和评审意见，再将这些文档交给编程智能体执行编码、测试和修复，必要时再引入评审智能体进行独立复核。

因此，当前项目仓库中的 `README.md`、`docs/文档导航.md`、`docs/project-memory.md`、`docs/specs/`、`docs/tasks/` 和 `docs/collab/` 承担了临时协作中枢的职责。所有外部智能体都应通过这些共同文档理解项目目标、当前阶段、职责边界和执行顺序。

智能软件工厂的核心价值，正是把这种依赖人工搬运上下文的协作方式产品化、系统化。系统建成后，每个数字员工都将从岗位模板实例化，自动获得岗位职责、技能资产、项目上下文、团队关系、运行时绑定和任务状态。智能体不再只是孤立工具，而是可被分配、可协作、可追踪、可治理的数字员工。

届时，项目负责人不再需要反复向每个智能体解释“你是谁、你负责什么、你该读哪些文档、你不能做什么”。这些信息将由智能软件工厂通过岗位、团队、项目、文档发布、决策闭环和运行时状态统一管理。人的角色也将从手动调度多个智能体，转变为管理智能体团队、处理关键决策和把控交付方向。

## 4. 核心对象

| 对象 | 定义 | 需求规则 |
|---|---|---|
| Skill | 可复用能力资产 | 包含名称、描述、版本、内容、状态、适用范围。 |
| SkillVersion | 技能内容真源 | 运行时初始化读取某个版本的快照。 |
| AgentTemplate | 岗位模板 | 定义岗位角色、职责、默认运行时类型和适用技能。 |
| TemplateSkillMapping | 岗位技能匹配 | 一个岗位可挂多个技能，一个技能可匹配多个岗位。 |
| Worker | 数字员工实例 | 从模板创建；不直接保存 skills 主数据。 |
| RuntimeHost | 运行时宿主 | 表达 OpenCode runtime 可部署在一台或多台机器。 |
| WorkerRuntimeBinding | 员工运行时绑定 | 记录 workspaceDir、skillDir、技能初始化快照与 syncStatus。 |
| AgentRoute | agentId 到 runtime endpoint 的内部路由 | 前端不可见 endpointRef；统一使用 routeStatus。 |
| AgentPresence | 在线/离线/忙碌/空闲状态 | 正式业务真源在 agent-team。 |
| Team | 团队 | 包含 Leader、成员、当前项目、活动流。 |
| Project | 项目 | P0a 阶段从 Team.currentProject 派生，不独立做多项目系统。 |
| Doc | 文档 | 支持 draft / published，发布是影响 Agent 上下文的动作。 |
| Decision | 待决策事项 | pending → resolved / blocked / expired。 |
| Activity | 活动事件 | 记录文档发布、决策处理、员工启停、技能同步等事件。 |

## 5. 功能分层

### 5.1 核心能力

必须进入 P0a：

```text
- Skill 资产展示与编辑模拟
- AgentTemplate / 岗位模板展示
- TemplateSkillMapping 展示和影响范围计算
- Worker 创建、启用、停用模拟
- RuntimeBinding 展示
- Team / Project 工作台
- Doc 保存草稿与发布模拟
- Decision 处理
- Activity 回写
```

### 5.2 连接能力

P0a 只做 mock：

```text
- 小云入口
- Leader target 切换
- MockChatAdapter
- hostContext 注入
- question / decision mock 事件
```

### 5.3 观察能力

P0a 做简化观察：

```text
- 总览指标
- 团队动态
- 员工状态
- RuntimeHost 与 WorkerRuntimeBinding 状态
- Route / Presence 摘要
```

## 6. 页面需求

### 6.1 总览页

展示：

```text
活跃团队数
在线员工数
待决策数
运行时状态摘要
团队动态流
团队协作拓扑或卡片视图
```

验收：

```text
- 能看到全部团队与 Leader。
- 能看到待分配员工或未启用员工。
- 点击团队可进入团队详情。
- 点击待决策可进入决策中心。
```

### 6.2 Skill 工作台

展示：

```text
Skill 列表
SkillVersion 当前版本
适用岗位
引用模板数
影响员工数
启用/停用状态
```

交互：

```text
- 新建/编辑 Skill 可先用本地状态模拟。
- 发布 SkillVersion 后生成 Activity。
- 修改 Skill 后展示影响范围：影响 N 个岗位、M 个员工、K 个 RuntimeBinding。
```

### 6.3 AgentTemplate / 岗位模板页

展示：

```text
模板名称
岗位职责
默认 runtimeType
匹配技能
由该模板创建的 Worker 数
```

交互：

```text
- 选择模板查看技能映射。
- 模拟从模板创建 Worker。
- 编辑模板技能映射后，派生员工技能视图更新。
```

### 6.4 Worker / 员工页

展示：

```text
员工名称
岗位模板
所属团队
生命周期状态
运行时状态
派生技能数量
RuntimeBinding syncStatus
当前任务摘要
```

规则：

```text
- 普通 Worker 不提供直接对话。
- 沟通入口必须表达为“通过 Leader 联系”。
- Worker.skills 由 domain selector 派生。
```

### 6.5 Runtime 监控页

展示：

```text
RuntimeHost 列表
host 在线/离线
绑定 Worker 数
workspaceDir 脱敏展示
AgentRoute routeStatus
AgentPresence presenceStatus
```

规则：

```text
- 前端 UI 不展示 endpointRef/token。
- workspaceDir 可作为运维摘要脱敏展示，但不传入聊天组件。
- disabled 不能被 heartbeat 自动恢复。
```

### 6.6 团队 / 项目页

展示：

```text
团队列表
团队 Leader
成员列表
当前项目
项目阶段
文档完成度
阻塞项
活动流
```

规则：

```text
- P0a 阶段一团队同一时间只有一个当前项目。
- Project 页是 Team.currentProject 的聚合视图。
```

### 6.7 项目文档库

展示：

```text
文档树
文档状态
草稿内容
发布版本
协作信息
历史摘要
```

交互：

```text
- 保存草稿只影响 draftContent。
- 发布文档生成 publishedContent / version。
- 发布后生成 Activity。
- 未来 Agent 读取 publishedContent，不读取未发布草稿。
```

### 6.8 决策中心

展示：

```text
待决策列表
决策上下文
候选方案
过期时间
处理状态
关联团队/项目/文档
```

交互：

```text
pending → resolved
pending → blocked
pending → expired
blocked → pending
```

处理后必须：

```text
- 更新状态。
- 写入 resolution。
- 生成 Activity。
- 从待处理提醒中移除。
```

---

## 7. P0a 验收闭环

P0a 完成时至少能演示：

```text
1. 查看 Skill 资产。
2. 查看 AgentTemplate 与技能映射。
3. 从模板创建 Worker。
4. Worker 派生技能正确展示。
5. Worker 绑定 RuntimeHost / workspaceDir mock 快照。
6. 启用 Worker 后 AgentRoute active、Presence online/idle。
7. 停用 Worker 后 routeStatus disabled，且不会被 heartbeat 自动恢复。
8. 团队中可以看到 Leader 与成员。
9. 项目文档可保存草稿并发布。
10. 发布文档生成 Activity。
11. 待决策可处理并生成 Activity。
12. 普通 Worker 沟通转到 Leader。
13. 小云使用 MockChatAdapter，不依赖真实 Gateway。
```
