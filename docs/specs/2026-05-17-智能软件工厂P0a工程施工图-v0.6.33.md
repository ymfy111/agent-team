> 版本：v0.6.33  
> 日期：2026-05-17  
> 状态：设计施工图 / 供 OpenCode 执行  
> 适用阶段：P0a 前端可演示 MVP  
> 约束：本文件只定义工程结构、模块职责、数据契约、执行顺序和验收边界；不要求 ChatGPT 直接编码，具体文件创建和编码交由 OpenCode 执行。

## 0. 本版本定位

v0.6.33 是从 v0.6.32 文档/原型基线进入 P0a 工程落地前的“施工图版本”。

v0.6.33 不改变产品大方向，不重新设计 PRD，不扩大 P0a 范围。它的目标是把已有设计细化到 OpenCode 可以按目录、模块、契约和验收项直接执行。

P0a 主线仍然是：

```text
Skill
→ AgentTemplate / 岗位模板
→ Worker / 数字员工实例 + RuntimeBinding
→ Team / Project
→ Doc 草稿 / 发布
→ Decision 处理
→ Activity 回写
```

P0a 不做真实后端、不做真实数据库、不接真实 agent-web-kit Gateway、不真实启动 OpenCode runtime。所有运行态、路由、心跳、会话和发布行为都以 mock seed + selectors + local state 模拟。

## 1. 设计原则

### 1.1 文档先于代码

OpenCode 开发前必须先读：

```text
README.md
docs/文档导航.md
docs/project-memory.md
docs/specs/2026-05-17-智能软件工厂产品需求规格-v0.6.33.md
docs/specs/2026-05-17-智能软件工厂系统设计方案-v0.6.33.md
docs/specs/2026-05-17-智能软件工厂Mock数据设计-v0.6.33.md
docs/tasks/2026-05-17-OpenCode执行清单-v0.6.33.md
docs/specs/2026-05-17-智能软件工厂P0a工程施工图-v0.6.33.md
```

若文档之间存在冲突，优先级为：

```text
v0.6.33 施工图
> v0.6.32 P0a 执行顺序清单
> v0.6.32 Mock 数据设计
> v0.6.32 系统设计
> v0.6.32 PRD
> 原型 HTML
```

### 1.2 先建模型，再做页面

禁止从页面视觉直接开工。P0a 的工程顺序必须是：

```text
contracts → domain → mock-seed → data-client → state/selectors → app/pages → ui polishing
```

任何页面只允许通过 data-client 和 selectors 读取数据，不允许页面内部散落造 mock 数据。

### 1.3 Skill 是主能力资产，Worker.skills 是派生视图

P0a 必须落实以下事实：

```text
Skill 是能力资产。
SkillVersion 是技能内容真源。
AgentTemplate 是岗位模板。
TemplateSkillMapping 是岗位-技能匹配真源。
Worker 从 AgentTemplate 创建。
Worker.skills 不是主数据，而是由 Skill + SkillVersion + TemplateSkillMapping + Worker.agentTemplateId 派生。
```

### 1.4 RuntimeBinding 是运行时快照，不是技能真源

一个 Worker 对应一个具体 Agent Runtime。P0a 中不真实启动 runtime，但必须在数据模型和 UI 中体现：

```text
WorkerRuntimeBinding = workerId + runtimeHostId + workspaceDir + skillDir + initializedSkillSnapshot + syncStatus
```

workspaceDir 中的技能文件只是运行时初始化快照，不是技能主数据。

### 1.5 前端 agentId-only

前端和聊天入口只允许传：

```text
agentId
conversationId
hostContext
```

不得暴露或传递：

```text
endpointRef
workspaceDir
runtimeHostId
OpenCode port
token
```

### 1.6 routeStatus 是唯一状态字段

AgentRoute 中只使用 `routeStatus`：

```text
active | disabled | unavailable | error
```

禁止新增或继续传播 `AgentRoute.status`。如果历史 mock 或适配数据中出现 `status`，必须在进入 domain / selector / DTO 前归一化为 `routeStatus`。

## 3. 运行时目录结构设计

P0a 不真实创建 OpenCode 工作目录，但 contracts、mock seed 和 UI 必须按未来真实目录建模。建议在设计中采用如下逻辑结构：

```text
<runtimeHost.rootDir>/
├── workers/
│   └── <workerId>/
│       ├── workspace/                  # OpenCode 工作目录，P0b/P1 真实使用
│       │   ├── AGENTS.md               # 运行时给该员工的项目级规则，P0a 仅展示路径
│       │   ├── skills/                 # 从 SkillVersion 初始化的运行时快照
│       │   │   ├── <skillSlug>/
│       │   │   │   ├── SKILL.md
│       │   │   │   ├── scripts/
│       │   │   │   └── resources/
│       │   ├── project/                # 可选：绑定项目代码目录或软链接
│       │   └── .agent-team/
│       │       ├── binding.json        # WorkerRuntimeBinding 快照
│       │       ├── initialized-skills.json
│       │       ├── sync-status.json
│       │       └── last-run.json
│       ├── logs/
│       └── tmp/
│
├── runtime-service/
│   ├── host-heartbeat.json
│   ├── agents-heartbeat.jsonl
│   └── supervisor.log
│
└── conversations/
    └── <conversationId>/
        ├── route-binding.json
        ├── messages.jsonl
        └── events.jsonl
```

P0a 只需要在 mock 数据中展示这些路径，不做真实文件写入。

### 3.1 WorkerRuntimeBinding 字段要求

```ts
type WorkerRuntimeBinding = {
  id: string
  workerId: string
  runtimeHostId: string
  runtimeType: 'opencode' | 'mock' | 'custom'
  workspaceDir: string
  skillDir: string
  initializedSkillSnapshot: Array<{
    skillId: string
    skillVersionId: string
    skillSlug: string
    initializedAt: string
    source: 'global' | 'template'
  }>
  syncStatus: 'synced' | 'pending' | 'out_of_sync' | 'failed' | 'not_initialized'
  lastSyncedAt?: string
  lastSyncError?: string
}
```

### 3.2 RuntimeHost 字段要求

```ts
type RuntimeHost = {
  id: string
  name: string
  hostType: 'local' | 'remote' | 'container' | 'mock'
  rootDir: string
  baseUrl?: string
  lifecycleStatus: 'enabled' | 'disabled'
  healthStatus: 'online' | 'offline' | 'degraded' | 'unknown'
  capacity?: {
    maxWorkers?: number
    runningWorkers?: number
  }
  lastHeartbeatAt?: string
}
```

### 3.3 Host offline 推导规则

RuntimeHost offline 时：

```text
- 其下 active route 的有效状态视为 unavailable。
- 其下 presenceStatus 的有效状态视为 offline。
- disabled route 不被覆盖，仍保持 disabled。
- Host 恢复 online 后，不批量自动恢复所有 route 为 active。
- Agent 必须等待自身 heartbeat 或业务动作恢复。
```

## 4. contracts 包设计

contracts 包是所有包的类型真源。禁止在页面或 mock 中重新定义重复类型。

### 4.1 ID 类型

```ts
type Brand<K, T> = K & { readonly __brand: T }

type SkillId = Brand<string, 'SkillId'>
type SkillVersionId = Brand<string, 'SkillVersionId'>
type AgentTemplateId = Brand<string, 'AgentTemplateId'>
type WorkerId = Brand<string, 'WorkerId'>
type RuntimeHostId = Brand<string, 'RuntimeHostId'>
type WorkerRuntimeBindingId = Brand<string, 'WorkerRuntimeBindingId'>
type AgentRouteId = Brand<string, 'AgentRouteId'>
type AgentId = Brand<string, 'AgentId'>
type TeamId = Brand<string, 'TeamId'>
type ProjectId = Brand<string, 'ProjectId'>
type DocId = Brand<string, 'DocId'>
type DocVersionId = Brand<string, 'DocVersionId'>
type DecisionId = Brand<string, 'DecisionId'>
type ActivityId = Brand<string, 'ActivityId'>
type ConversationId = Brand<string, 'ConversationId'>
```

P0a 可以用普通字符串实现，但类型层面建议预留 Brand，减少误传。

### 4.2 Skill / SkillVersion

```ts
type SkillStatus = 'draft' | 'published' | 'disabled' | 'archived'

type Skill = {
  id: SkillId
  slug: string
  name: string
  description: string
  category: 'engineering' | 'product' | 'design' | 'qa' | 'release' | 'security' | 'general'
  status: SkillStatus
  currentVersionId?: SkillVersionId
  createdAt: string
  updatedAt: string
}

type SkillVersion = {
  id: SkillVersionId
  skillId: SkillId
  version: string
  content: string
  assets?: {
    scripts?: Array<{ path: string; description?: string }>
    resources?: Array<{ path: string; description?: string }>
  }
  changelog?: string
  createdAt: string
  publishedAt?: string
  createdBy?: string
}
```

### 4.3 AgentTemplate / TemplateSkillMapping

```ts
type AgentTemplateRole = 'leader' | 'architect' | 'developer' | 'tester' | 'designer' | 'modeler' | 'custom'

type AgentTemplate = {
  id: AgentTemplateId
  code: string
  name: string
  role: AgentTemplateRole
  description: string
  defaultProjectRole?: string
  lifecycleStatus: 'enabled' | 'disabled'
  createdAt: string
  updatedAt: string
}

type TemplateSkillMapping = {
  id: string
  agentTemplateId: AgentTemplateId | '*'
  skillId: SkillId
  source: 'global' | 'template'
  required: boolean
  order: number
  createdAt: string
}
```

约束：

```text
agentTemplateId = '*' 表示通用技能。
同一个 template + skill 只能存在一条有效 mapping。
required=true 的技能在 P0a 中不可从模板移除，只能禁用 Skill 或创建新模板。
```

### 4.4 Worker

```ts
type WorkerLifecycleStatus = 'draft' | 'enabled' | 'disabled' | 'archived'
type WorkerRuntimeStatus = 'not_initialized' | 'starting' | 'running' | 'idle' | 'busy' | 'stopping' | 'disabled' | 'failed' | 'offline'

type Worker = {
  id: WorkerId
  agentId: AgentId
  name: string
  agentTemplateId: AgentTemplateId
  teamId?: TeamId
  projectId?: ProjectId
  projectRole?: string
  lifecycleStatus: WorkerLifecycleStatus
  runtimeStatus: WorkerRuntimeStatus
  currentTaskSummary?: string
  createdAt: string
  updatedAt: string
}
```

注意：Worker 不保存 skills 数组，最多在 view model 中展示 `derivedSkills`。

### 4.5 AgentRoute / AgentPresence

```ts
type AgentRoute = {
  id: AgentRouteId
  agentId: AgentId
  ownerType: 'global' | 'leader' | 'worker'
  ownerId: string
  routeRevision: number
  routeStatus: 'active' | 'disabled' | 'unavailable' | 'error'
  runtimeType: 'opencode' | 'mock' | 'custom'
  runtimeHostId?: RuntimeHostId
  endpointRef: string
  workspaceDir?: string
  capabilities?: string[]
  updatedAt: string
}

type AgentPresence = {
  agentId: AgentId
  runtimeHostId?: RuntimeHostId
  routeStatus: 'active' | 'disabled' | 'unavailable' | 'error'
  presenceStatus: 'online' | 'offline' | 'busy' | 'idle' | 'starting' | 'stopping' | 'failed'
  currentConversationId?: ConversationId
  currentTaskSummary?: string
  lastHeartbeatAt?: string
  updatedAt: string

```

routeRevision 递增规则：

```text
必须递增：endpointRef、runtimeHostId、workspaceDir、runtimeType、routeStatus、影响会话能力的 capabilities 变化。
不应递增：lastHeartbeatAt、presenceStatus、currentTaskSummary、metrics 变化。
```

### 4.6 ConversationRouteBinding

```ts
type ConversationRouteBinding = {
  conversationId: ConversationId
  agentId: AgentId
  routeRevision: number
  runtimeType: 'opencode' | 'mock' | 'custom'

```

P0a 中只做 mock 展示和聊天适配层内部数据，不暴露给浏览器页面。

## 5. domain 包设计

### 5.1 必须实现的派生规则

#### 5.1.1 deriveWorkerSkills

输入：

```text
Worker
AgentTemplate
Skill[]
SkillVersion[]
TemplateSkillMapping[]
```

输出：

```text
DerivedWorkerSkill[]
```

规则：

```text
1. 找到 Worker.agentTemplateId 对应模板。
2. 取 agentTemplateId='*' 的通用 mapping。
3. 取 agentTemplateId=当前模板的模板 mapping。
4. 过滤 disabled / archived Skill。
5. 每个 Skill 使用 currentVersionId 指向的 SkillVersion。
6. 按 source(global/template) + order 排序。
7. 输出 source、required、skillVersionId、contentPreview。
```

#### 5.1.2 computeSkillImpact

用于技能编辑/发布前的影响范围提示：

```text
输入：skillId
输出：
- affectedTemplateIds
- affectedWorkerIds
- affectedRuntimeBindingIds
- needRuntimeSyncWorkerIds
```

P0a 中保存草稿不改变影响范围；发布新版本后，相关 WorkerRuntimeBinding 的 syncStatus 可模拟变为 `out_of_sync` 或 `pending`。

#### 5.1.3 computeRuntimeSyncStatus

同步状态来源：

```text
- WorkerRuntimeBinding.initializedSkillSnapshot
- 当前模板派生出来的 SkillVersion 集合
- RuntimeHost.healthStatus
- Worker.lifecycleStatus
```

规则：

```text
Worker disabled → syncStatus 不自动改变，但 UI 禁止同步动作。
Host offline → 显示无法同步，不把 syncStatus 改为 failed。
派生技能版本与 initializedSkillSnapshot 完全一致 → synced。
派生技能包含新版本但未初始化 → out_of_sync。
缺少必要技能快照 → not_initialized。
最近一次同步动作失败 → failed。
```

#### 5.1.4 deriveEffectiveRoute

规则：

```text
route.routeStatus = disabled → effectiveRouteStatus = disabled。
route.routeStatus = error → effectiveRouteStatus = error。
route.routeStatus = active 且 RuntimeHost offline → effectiveRouteStatus = unavailable。
route.routeStatus = active 且 RuntimeHost online → effectiveRouteStatus = active。
```

#### 5.1.5 deriveEffectivePresence

规则：

```text
Worker.lifecycleStatus = disabled → presenceStatus = offline 或 disabled-view。
RuntimeHost offline → effectivePresenceStatus = offline。
AgentPresence.presenceStatus = busy/idle/online → 按原值展示。
会话级 busy 不能直接覆盖员工级 busy。
```

#### 5.1.6 canWorkerChat

规则：

```text
Leader：可直接打开小云 target。
普通 Worker：不能直接对话，只能通过所属 Team Leader 联系。
未入组 Worker：不能对话，可展示“需先加入团队”。
routeStatus disabled/unavailable/error：禁止新会话。
```
