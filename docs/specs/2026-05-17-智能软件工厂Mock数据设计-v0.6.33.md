# 智能软件工厂 Mock 数据设计

> 版本：v0.6.33  
> 日期：2026-05-17  
> 状态：当前正本 / P0a 唯一初始数据源设计

---

## 1. 设计目标

P0a 不接真实后端和数据库，因此 mock seed 必须承担“可演示业务闭环”的初始化数据职责。

目标：

```text
1. 支撑 Skill → AgentTemplate → Worker → RuntimeBinding 派生关系。
2. 支撑 Team / Project / Doc / Decision / Activity 页面展示。
3. 支撑 Worker 启用、停用、技能同步状态模拟。
4. 支撑 route/presence 状态推导。
5. 保证页面不再各自散落 mock 数据。
```

## 3. 建议数据规模

| 类型 | 数量 | 说明 |
|---|---:|---|
| RuntimeHost | 3 | local/mock/remote-degraded，用于展示多宿主和离线推导。 |
| Skill | 10-16 | 包含通用技能、架构、开发、测试、设计、建模、Leader 技能。 |
| AgentTemplate | 6 | Leader、架构、开发、测试、设计、建模。 |
| Worker | 20-30 | 5 个团队 + 待分配池 + disabled 示例。 |
| Team | 5 | 研发一组到研发五组。 |
| Project | 5 | 每组一个当前项目。 |
| Doc | 15-25 | specs/plans/decisions/reports/notes 分类。 |
| Decision | 6-10 | pending/resolved/blocked/expired 都要覆盖。 |
| Activity | 20+ | 覆盖技能、员工、文档、决策、运行时事件。 |

## 4. 必须覆盖的场景

### 4.1 技能派生

```text
- 通用技能适用于所有模板。
- 开发模板拥有开发专属技能。
- 测试模板拥有 QA 专属技能。
- Leader 拥有协调、决策、文档汇总技能。
- disabled Skill 不进入 Worker 派生技能。
```

验收：

```text
deriveWorkerSkills(workerId) 能输出来源：global / template。
```

### 4.2 技能影响范围

```text
修改一个通用技能 → 影响所有模板和所有 Worker。
修改一个岗位技能 → 影响对应模板和由该模板创建的 Worker。
停用一个技能 → 影响范围仍可计算，但派生视图中不再展示为可用技能。
```

### 4.3 RuntimeHost 状态

必须至少包含：

```text
host-local-1：online
host-local-2：online/degraded
host-remote-1：offline
```

推导规则：

```text
RuntimeHost offline → 其下 active route 视为 unavailable。
RuntimeHost offline → 其下 presenceStatus 视为 offline。
RuntimeHost offline 不覆盖 disabled route。
```

### 4.4 WorkerRuntimeBinding syncStatus

必须覆盖：

```text
synced
pending
out_of_sync
failed
not_initialized
```

示例：

```text
某 Worker 的 SkillVersion 已更新，但 initializedSkillSnapshot 仍是旧版本 → out_of_sync。
某 Worker 从模板创建但尚未启用 → not_initialized。
某 RuntimeHost offline 的 Worker 不直接标 failed，而是通过 host 状态推导不可用。
```

### 4.5 AgentRoute / Presence

必须覆盖：

```text
active + online/idle
active + busy
disabled + offline
unavailable + offline
error + failed
```

禁止 seed 中出现：

```text
AgentRoute.status
```

### 4.6 文档发布

必须有：

```text
draft-only 文档
已发布文档
发布后又有新草稿的文档
关联决策的文档
```

### 4.7 决策闭环

必须有：

```text
pending 紧急决策
pending 普通决策
resolved 决策
blocked 决策
expired 决策
```

处理 pending 决策后必须能生成 Activity。

## 7. 禁止事项

```text
- 不允许页面自己造 mock 数据。
- 不允许 mock seed 里存 Worker.skills 主字段。
- 不允许出现 AgentRoute.status。
- 不允许把 endpointRef/token 作为前端可见字段。
- 不允许把 Gateway 状态当作业务状态真源。
```
