# 智能软件工厂 Mock 数据设计

> 版本：v0.6.32  
> 日期：2026-05-15  
> 状态：P0a 开工准备稿  
> 关联：系统设计 v0.6.32、开发计划 v0.6.32、开发任务拆解 v0.6.32。

## 1. 设计目标

P0a 使用 mock seed、本地状态和 MockChatAdapter 跑通最小业务闭环。本文定义 P0a 初始数据、对象关系和 ID 规则，避免不同页面各自造数据。

P0a 主线：

```text
Skill → AgentTemplate → Worker + RuntimeBinding → Team/Project → Doc 发布 → Decision 处理 → Activity 回写
```

## 2. 数据设计原则

1. mock seed 是 P0a 唯一初始数据源。
2. `SkillVersion` 和 `TemplateSkillMapping` 是技能继承和发布生效的依据。
3. `Worker.skills[]` 不作为主数据，只由模板技能匹配和 SkillVersion 派生。
4. 每个 `Worker` 都应有一个 `WorkerRuntimeBinding`。
5. `workspaceDir` 内的 skill 文件是运行时快照，不反向覆盖 Skill 资产库。
6. P0a 不真实写文件、不真实连接 OpenCode、不做真实权限和审计。

## 3. 顶层 mock seed 结构

```ts
type MockSeed = {
  runtimeHosts: RuntimeHost[]
  agentTemplates: AgentTemplate[]
  skills: Skill[]
  skillVersions: SkillVersion[]
  templateSkillMappings: TemplateSkillMapping[]
  workers: Worker[]
  workerRuntimeBindings: WorkerRuntimeBinding[]
  agentRoutes: AgentRoute[]
  agentPresences: AgentPresence[]
  teams: Team[]
  projects: Project[]
  projectAssignments: ProjectAssignment[]
  docs: Doc[]
  docVersions: DocVersion[]
  decisions: Decision[]
  activities: Activity[]
  mockCurrentUser: MockUser
}
```

## 4. RuntimeHost

P0a 至少提供一个默认运行节点，表达“当前所有 Worker 可先跑在同一台 OpenCode Host 上”；同时模型应支持未来多节点。

```ts
type RuntimeHost = {
  id: string
  name: string
  runtimeType: 'opencode'
  endpoint?: string
  workspaceRoot: string
  status: 'online' | 'offline' | 'degraded'
  capacity?: {
    maxWorkers?: number
    activeWorkers?: number
  }
  labels?: string[]
}
```

示例：

```ts
const runtimeHosts = [
  {
    id: 'rh-local-001',
    name: '本地 OpenCode 节点',
    runtimeType: 'opencode',
    endpoint: 'http://127.0.0.1:7000',
    workspaceRoot: '/workspace/agent-team/workers',
    status: 'online',
    capacity: { maxWorkers: 20, activeWorkers: 8 },
    labels: ['local', 'linux', 'p0a-mock'],
  },
]
```

## 5. AgentTemplate

```ts
type AgentTemplate = {
  id: string
  code: 'explorer' | 'fixer' | 'oracle' | 'designer'
  displayName: string
  description: string
  responsibilities: string[]
  enabled: boolean
}
```

建议初始模板：

| id | code | UI 名称 | 说明 |
|---|---|---|---|
| `tpl-explorer` | explorer | 测试工程师 | 需求理解、用例、验证与质量反馈 |
| `tpl-fixer` | fixer | 定制开发 | 代码实现、缺陷修复、联调 |
| `tpl-oracle` | oracle | 架构师 | 架构判断、技术路线、风险分析 |
| `tpl-designer` | designer | 设计师 | 原型、交互、界面评审 |

## 6. Skill 与 SkillVersion

```ts
type Skill = {
  id: string
  name: string
  description: string
  status: 'draft' | 'published' | 'disabled'
  scope: 'global' | 'template'
  draftVersionId?: string
  publishedVersionId?: string
}

type SkillVersion = {
  id: string
  skillId: string
  version: number
  status: 'draft' | 'published'
  frontmatter: Record<string, unknown>
  markdownContent: string
  files: Array<{ path: string; kind: 'skill-md' | 'script' | 'resource' }>
  createdAt: string
  publishedAt?: string
}
```

建议至少包含：

| Skill | scope | 说明 |
|---|---|---|
| 通用任务拆解 | global | 所有数字员工均可用 |
| 文档阅读与摘要 | global | 读取项目文档并提炼上下文 |
| 代码实现 | template | fixer 专属 |
| 架构评审 | template | oracle 专属 |
| 原型走查 | template | designer 专属 |
| 测试用例生成 | template | explorer 专属 |

## 7. TemplateSkillMapping

```ts
type TemplateSkillMapping = {
  agentTemplateId: string
  skillId: string
  source: 'global' | 'template'
  enabled: boolean
  effectiveVersion?: number
}
```

规则：

- global skill 可以用 mapping 显式挂给所有模板，也可以由 selector 统一派生。
- template skill 必须通过 TemplateSkillMapping 挂到对应 AgentTemplate。
- Worker.skills[] 由 `worker.agentTemplateId + mappings + skillVersions` 派生。

## 8. Worker 与 WorkerRuntimeBinding

```ts
type Worker = {
  id: string
  name: string
  agentTemplateId: string
  projectRole: string
  teamId?: string
  projectId?: string
  status: 'idle' | 'busy' | 'offline'
  currentTaskSummary?: string
  runtimeBindingId: string
  heartbeatTs?: string
}

type WorkerRuntimeBinding = {
  id: string
  workerId: string
  runtimeType: 'opencode'
  runtimeHostId: string
  workspaceDir: string
  skillDir: string
  initializedSkillVersionRefs: Array<{
    skillId: string
    skillVersion: number
    source: 'global' | 'template'
    initializedAt: string
  }>
  syncStatus: 'not_initialized' | 'synced' | 'outdated' | 'sync_failed'
  lastSyncAt?: string
  lastError?: string
}
```

示例规则：

```text
worker.id = worker-dev-1
runtimeHostId = rh-local-001
workspaceDir = /workspace/agent-team/workers/worker-dev-1
skillDir = /workspace/agent-team/workers/worker-dev-1/skills
syncStatus = synced
```

## 9. 技能初始化快照

创建 Worker 时，根据模板解析出的已发布 SkillVersion 写入 `initializedSkillVersionRefs`。

示例：

```ts
initializedSkillVersionRefs: [
  { skillId: 'skill-common-task-breakdown', skillVersion: 1, source: 'global', initializedAt: '2026-05-15T09:00:00Z' },
  { skillId: 'skill-fixer-code-implementation', skillVersion: 2, source: 'template', initializedAt: '2026-05-15T09:00:00Z' },
]
```

当 Skill 发布新版本时：

- P0a：只把受影响 Worker 的 `syncStatus` 模拟为 `outdated` 或在点击“发布生效”后模拟刷新为 `synced`。
- P0b：可在单机 RuntimeHost 上写入新版本文件并更新快照。
- P1：支持多 RuntimeHost 批量同步、失败重试、差异查看。

## 10. Team / Project / Assignment

P0a 继续保持：一个 Team 同一时间承接一个当前 Project。

```ts
type ProjectAssignment = {
  id: string
  workerId: string
  teamId: string
  projectId: string
  projectRole: string
}
```

要求：

- Worker 加入团队后，团队页、项目页、总览读侧必须同步。
- Worker 的 projectRole 是项目职责帽子，不等同 AgentTemplate。

## 11. Doc / Decision / Activity

P0a 至少包含：

- 1 个可编辑并发布的需求文档。
- 1 个可处理的 pending Decision。
- 多条 Activity，覆盖 Skill 发布、员工创建、文档发布、决策处理。

Activity 示例：

```ts
{
  id: 'act-worker-runtime-init-001',
  teamId: 'team-1',
  type: 'worker_runtime_initialized',
  desc: '基于“定制开发”模板创建开发1-3，并在本地 OpenCode 节点初始化 4 个 Skill。',
  createdAt: '2026-05-15T09:10:00Z',
}
```

## 13. P0a 验收点

| 验收点 | 期望 |
|---|---|
| mock seed 初始化 | 所有页面从同一份 seed 派生数据 |
| Worker 运行时绑定 | 每个 Worker 都能展示 RuntimeHost、workspaceDir、skillDir、syncStatus |
| 技能继承 | Worker 技能由模板和 SkillVersion 派生，不独立维护 |
| Skill 发布 | 影响范围可计算，Worker runtime syncStatus 可模拟更新 |
| 多机预留 | seed 结构允许新增第二个 RuntimeHost，不改 Worker 主结构 |



## 8. AgentRoute 与 AgentPresence mock

P0a 不连接真实 Gateway，但 mock seed 需要表达路由和运行态，方便员工页、团队页和 MockChatAdapter 使用统一状态。

```ts
const agentRoutes: AgentRoute[] = [
  {
    agentId: 'leader-team-001',
    ownerType: 'leader',
    ownerId: 'worker-leader-001',
    runtimeType: 'opencode',
    runtimeHostId: 'runtime-local-001',
    endpointRef: 'mock://leader-team-001',
    workspaceDir: '/mock/runtime/workers/leader-team-001',
    routeStatus: 'active',
    routeRevision: 1,
    updatedAt: '2026-05-15T10:00:00Z',
  },
  {
    agentId: 'worker-dev-001',
    ownerType: 'worker',
    ownerId: 'worker-dev-001',
    runtimeType: 'opencode',
    runtimeHostId: 'runtime-local-001',
    endpointRef: 'mock://worker-dev-001',
    workspaceDir: '/mock/runtime/workers/worker-dev-001',
    routeStatus: 'disabled',
    routeRevision: 1,
    updatedAt: '2026-05-15T10:00:00Z',
  },
]
```

```ts
const agentPresences: AgentPresence[] = [
  {
    agentId: 'leader-team-001',
    runtimeHostId: 'runtime-local-001',
    routeStatus: 'active',
    presenceStatus: 'idle',
    lastHeartbeatAt: '2026-05-15T10:00:00Z',
    updatedAt: '2026-05-15T10:00:00Z',
  },
  {
    agentId: 'worker-dev-001',
    runtimeHostId: 'runtime-local-001',
    routeStatus: 'disabled',
    presenceStatus: 'offline',
    updatedAt: '2026-05-15T10:00:00Z',
  },
]
```

规则：

- `disabled` 员工在 mock 中只允许重新启用。
- `busy` 显示为员工级忙碌，不等同于小云对话框 streaming。
- RuntimeHost 离线时，可将该 host 下所有 AgentPresence 派生为 offline。

## 补充：AgentRoute / AgentPresence mock 字段收敛

P0a mock seed 中的 AgentRoute 统一使用 `routeStatus`，不再使用模糊的 `status` 字段。

```ts
type MockAgentRoute = {
  agentId: string
  ownerType: 'global' | 'leader' | 'worker'
  ownerId: string
  routeRevision: number
  routeStatus: 'active' | 'disabled' | 'unavailable' | 'error'
  runtimeType: 'opencode' | 'mock' | 'custom'
  runtimeHostId?: string
  endpointRef: string
  workspaceDir?: string
  capabilities?: string[]
}
```

P0a 可以 mock `ConversationRouteBinding`，但不做真实 Gateway 会话绑定：

```ts
type MockConversationRouteBinding = {
  conversationId: string
  agentId: string
  routeRevision: number
  runtimeType: 'opencode' | 'mock' | 'custom'
  runtimeHostId?: string
  endpointRef: string
  workspaceDir?: string
  boundAt: string
}
```

P0a 状态模拟原则：

```text
- routeStatus = disabled 的员工，只显示“重新启用”。
- disabled 不能被 mock heartbeat 自动恢复为 active。
- RuntimeHost offline 时，其下 active route 在 selector 中显示为 unavailable。
- RuntimeHost 恢复 online 后，不自动批量恢复 Agent；需要单个 Agent mock 心跳或状态切换。
```
