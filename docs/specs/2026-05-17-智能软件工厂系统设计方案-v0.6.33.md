## 1. 总体架构

P0a 采用前端 monorepo 架构，暂不包含真实后端。

```text
agent-team/
├── apps/web                    # React/Vite 前端应用
└── packages/
    ├── contracts               # 类型契约
    ├── domain                  # 纯业务规则和 selectors
    ├── mock-seed               # 唯一初始 mock 数据源
    ├── data-client             # mockClient / future API client 抽象
    ├── state                   # 前端状态和 commands
    ├── ui                      # UI primitives
    ├── chat-integration        # 小云 / agent-web-kit 适配边界
    └── testing                 # 未来测试工具
```

依赖方向：

```text
contracts
  ↓
domain ─── mock-seed
  ↓          ↓
data-client → state → apps/web
ui ────────────────┘
chat-integration ──┘
```

禁止反向依赖：

```text
contracts 不依赖任何业务实现。
domain 不依赖 React、浏览器、网络。
mock-seed 不依赖页面。
页面不直接造 mock 数据。
chat-integration 不拥有团队/项目/文档主数据。
```

## 2. 核心域模型

### 2.1 Skill

```ts
type Skill = {
  id: SkillId
  slug: string
  name: string
  description: string
  status: 'enabled' | 'disabled'
  currentVersionId: SkillVersionId
  createdAt: string
  updatedAt: string
}
```

```ts
type SkillVersion = {
  id: SkillVersionId
  skillId: SkillId
  version: string
  content: string
  usage?: string
  inputContract?: string
  outputContract?: string
  toolPolicy?: string
  verification?: string
  assets?: {
    scripts?: Array<{ path: string; description?: string }>
    resources?: Array<{ path: string; description?: string }>
  }
  createdAt: string
}
```

设计规则：

```text
Skill 是能力资产。
SkillVersion 是内容真源。
运行时目录里的 SKILL.md 是初始化快照，不是主数据。
```

### 2.2 AgentTemplate

```ts
type AgentTemplate = {
  id: AgentTemplateId
  roleKey: 'leader' | 'architect' | 'developer' | 'tester' | 'designer' | 'modeler'
  name: string
  description: string
  defaultRuntimeType: 'mock' | 'opencode' | 'custom'
  lifecycleStatus: 'draft' | 'active' | 'deprecated'
  createdAt: string
  updatedAt: string
}
```

```ts
type TemplateSkillMapping = {
  id: string
  templateId: AgentTemplateId | '*'
  skillId: SkillId
  required: boolean
  order: number
}
```

规则：

```text
templateId='*' 表示通用技能。
同一模板的所有 Worker 共享同一组派生技能。
```

### 2.3 Worker

```ts
type Worker = {
  id: WorkerId
  agentId: AgentId
  name: string
  templateId: AgentTemplateId
  teamId?: TeamId
  projectRole?: string
  lifecycleStatus: 'draft' | 'enabled' | 'disabled' | 'archived'
  runtimeStatus: 'not_initialized' | 'starting' | 'running' | 'stopping' | 'stopped' | 'failed' | 'disabled'
  currentTaskSummary?: string
  createdAt: string
  updatedAt: string
}
```

Worker 不保存 `skills[]` 主字段。页面展示的技能由 `deriveWorkerSkills()` 计算。

### 2.4 RuntimeHost 与 WorkerRuntimeBinding

```ts
type RuntimeHost = {
  id: RuntimeHostId
  name: string
  hostKind: 'local' | 'remote' | 'container' | 'mock'
  status: 'online' | 'offline' | 'degraded'
  rootDir?: string
  lastHeartbeatAt?: string
}
```

```ts
type WorkerRuntimeBinding = {
  id: string
  workerId: WorkerId
  runtimeHostId: RuntimeHostId
  runtimeType: 'mock' | 'opencode' | 'custom'
  workspaceDir: string
  skillDir: string
  initializedSkillSnapshot: Array<{
    skillId: SkillId
    skillVersionId: SkillVersionId
    path: string
    checksum?: string
    initializedAt: string
  }>
  syncStatus: 'not_initialized' | 'synced' | 'pending' | 'out_of_sync' | 'failed'
  lastSyncAt?: string
}
```

逻辑目录：

```text
<runtimeHost.rootDir>/
├── workers/<workerId>/workspace/
│   ├── AGENTS.md
│   ├── skills/<skillSlug>/SKILL.md
│   └── .agent-team/
│       ├── binding.json
│       ├── initialized-skills.json
│       └── sync-status.json
└── conversations/<conversationId>/route-binding.json
```

P0a 只展示这些字段和目录心智，不真实写入文件系统。

## 3. Route / Presence 设计

### 3.1 AgentRoute

```ts
type AgentRoute = {
  agentId: AgentId
  ownerType: 'global' | 'leader' | 'worker'
  ownerId: string
  routeRevision: number
  routeStatus: 'active' | 'disabled' | 'unavailable' | 'error'
  runtimeType: 'opencode' | 'mock' | 'custom'
  runtimeHostId?: RuntimeHostId
  endpointRef: string
  workspaceDir?: string
  capabilities?: string[]
}
```

规则：

```text
- routeStatus 是唯一状态字段。
- status 字段不得进入 domain / selector / DTO。
- endpointRef、workspaceDir、runtimeHostId 不暴露给浏览器聊天组件。
```

### 3.2 AgentPresence

```ts
type AgentPresence = {
  agentId: AgentId
  runtimeHostId?: RuntimeHostId
  routeStatus: 'active' | 'disabled' | 'unavailable' | 'error'
  presenceStatus: 'online' | 'offline' | 'busy' | 'idle' | 'starting' | 'stopping' | 'failed'
  currentConversationId?: ConversationId
  currentTaskSummary?: string
  lastHeartbeatAt?: string
  updatedAt: string
}
```

### 3.3 routeRevision

必须递增：

```text
endpointRef 变化
runtimeHostId 变化
workspaceDir 变化
runtimeType 变化
routeStatus 从 active 变为 disabled/unavailable/error
capabilities 发生影响会话能力的变化
```

不应单独递增：

```text
lastHeartbeatAt 更新
presenceStatus 更新
currentTaskSummary 更新
metrics 更新
```

### 3.4 ConversationRouteBinding

```ts
type ConversationRouteBinding = {
  conversationId: ConversationId
  agentId: AgentId
  routeRevision: number
  runtimeType: 'opencode' | 'mock' | 'custom'
  runtimeHostId?: RuntimeHostId
  endpointRef: string
  workspaceDir?: string
  boundAt: string
}
```

规则：

```text
新 conversation 使用最新 AgentRoute。
已有 conversation 默认继续使用创建时的 snapshot。
routeStatus=disabled 后拒绝新 conversation。
ConversationRouteBinding 是后端/Gateway 内部数据，不暴露给浏览器前端。
```

## 4. 状态真源边界

正式业务系统：

```text
RuntimeHost / OpenCode supervisor
→ agent-team runtime-service
→ agent-team 数据库
→ agent-gateway Provider 查询路由和脱敏状态
→ widget 只拿 agentId 可见状态摘要
```

轻量 demo：

```text
Mock runtime
→ agent-gateway heartbeat
→ Gateway 自维护临时 route/presence
```

结论：

```text
Gateway 支持 heartbeat，但智能软件工厂正式集成时，业务状态真源是 agent-team。
```

## 5. 关键业务流

### 5.1 创建 Worker

```text
选择 AgentTemplate
→ 创建 Worker
→ 通过 TemplateSkillMapping 派生技能预览
→ 分配 Team / ProjectRole
→ 生成 Activity
```

### 5.2 启用 Worker

```text
选择或自动分配 RuntimeHost
→ 创建/复用 workspaceDir
→ 查模板匹配的 SkillVersion
→ 初始化技能快照到 WorkerRuntimeBinding
→ 激活 AgentRoute
→ AgentPresence starting/idle
→ Worker.lifecycleStatus=enabled
→ Worker.runtimeStatus=running
→ 生成 Activity
```

### 5.3 停用 Worker

```text
停止 runtime mock 状态
→ workspaceDir 不删除
→ initializedSkillSnapshot 不删除
→ AgentRoute.routeStatus=disabled
→ Worker.lifecycleStatus=disabled
→ Worker.runtimeStatus=disabled
→ Presence offline/stopping
→ 生成 Activity
```

停用后禁止：

```text
重启
下线
同步技能
对话
```

除非重新启用。

### 5.4 文档发布

```text
编辑 draftContent
→ 保存草稿
→ 发布文档
→ 写 publishedContent / publishedVersion
→ 生成 Activity
→ Agent 后续读取发布版本
```

### 5.5 决策处理

```text
Decision.pending
→ 用户选择方案
→ 写入 resolution
→ status=resolved
→ 生成 Activity
→ 更新提醒数量
```

## 6. P0a 实现边界

P0a 使用 mock：

```text
mock-seed = 唯一初始数据源
data-client/mockClient = 唯一读取入口
state commands = 唯一写操作入口
domain selectors = 唯一派生规则入口
```

禁止：

```text
页面直接 import seed 后自行过滤
页面内部造 Worker/Team/Decision mock
页面绕过 domain 计算 Worker.skills
页面把 status 当 routeStatus 使用
页面把 endpointRef/workspaceDir/token 传给聊天组件
```
