## 1. P0a 目标

P0a 是前端可演示 MVP，不接真实后端、真实数据库、真实 agent-web-kit Gateway、真实 OpenCode Runtime、真实 agent-browser。

P0a 要通过 mock seed 和前端状态演示：

```text
AI 原生岗位
→ 数字员工团队
→ Leader 协调
→ 任务单派发
→ 进度/阻塞/分歧
→ 待决策
→ 交付审查
→ 整改任务单
→ Activity 回写
```

```text
1. workspace 基础文件
```

## 5. domain 最小规则

第一轮至少实现这些 selector / rule：

```text
selectTeamLeader(teamId)
selectTeamMembers(teamId)
selectWorkersByTemplate(templateId)
selectWorkOrdersByTeam(teamId)
selectActiveWorkOrdersByWorker(workerId)
selectTeamProgressSummary(teamId)
selectBlockingItems(teamId)
selectPendingDecisions(teamId)
selectPendingAcceptances(teamId)
```

规则要求：

```text
1. Leader 必须从 team.leaderWorkerId 得出，不从岗位名硬编码。
2. Worker.skills 必须由 AgentTemplate + TemplateSkillMapping + SkillVersion 派生。
3. 实现验证岗的 self_check 不等于交付验收通过。
4. acceptanceStrategy = independent_acceptance 时，验收 Worker 不应等于执行 Worker。
5. routeStatus 使用 routeStatus，不新增 AgentRoute.status。
6. 前端组件不得读取 endpointRef / workspaceDir / token。
```

## 6. mock-seed 要求

mock seed 必须体现：

```text
3 个 AgentTemplate：协同规划岗、实现验证岗、交付审查岗。
至少 1 个团队：规划1 为 Leader，实现1-1/实现1-2 为成员，审查1 为成员。
至少 3 个 WorkOrder：进行中、阻塞、待验收。
至少 1 个 Decision：Leader 无法裁决，上报用户。
至少 1 个 AcceptanceRecord：交付审查不通过并生成整改建议。
至少 1 个 BrowserRuntimeSession：agent-browser provider，占位状态。
```

## 7. 页面实现优先级

P0a 页面不是第一轮重点，但后续页面应按以下顺序实现：

```text
1. AppShell / 主导航
2. 总览页：团队、任务、阻塞、待验收、待决策总览
3. 团队工作台：Leader、成员、当前任务、进度、阻塞、活动
4. 员工页：按 AI 原生岗位筛选 Worker
5. 待决策页：Leader 上报给用户的问题
6. 文档库：需求、设计、任务单、变更、验收记录
7. 交付审查视图：验收结论、整改建议、证据
```

## 8. 第一轮任务边界

第一轮只允许：

```text
- 创建 workspace 基础文件。
- 创建 contracts 包。
- 创建 domain 包。
- 建立最小类型和 selector。
- 写最小测试或 typecheck 脚本。
```

禁止：

```text
- 写完整页面。
- 接真实后端。
- 接真实 Gateway。
- 接真实 OpenCode Runtime。
- 接真实 agent-browser。
- 新增旧岗位模型。
- 恢复 v0.6.32 旧原型或旧 mock。
```
