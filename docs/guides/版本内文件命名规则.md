# 版本内文件命名规则

> 适用项目：智能软件工厂 / agent-team
> 当前版本：v0.6.32

## 1. 规则

同一个版本内，当前正本文件名不再使用 `-revised`、`-p0a-flow`、`-ui-review-task-breakdown` 等表示先后顺序的后缀。

版本内的小改、评审修订和原型增强通过 Git commit 追踪，不通过文件名后缀区分。

## 2. 当前正本命名

```text
docs/specs/2026-05-13-智能软件工厂产品需求规格-v0.6.32.md
docs/specs/2026-05-13-智能软件工厂系统设计方案-v0.6.32.md
docs/specs/2026-05-14-智能软件工厂-agent-web-kit集成方案-v0.6.32.md
docs/prototypes/Agent-Team-V2-注册中心-v0.6.32.html
docs/plans/2026-05-14-智能软件工厂开发计划-v0.6.32.md
docs/tasks/2026-05-14-智能软件工厂开发任务拆解-v0.6.32.md
```

## 3. 例外

报告、ADR、变更说明可以保留主题名，因为它们不是同一类文档的当前正本。

例如：

```text
docs/reports/2026-05-15-v0.6.32文档-原型一致性评审报告.md
docs/reports/2026-05-15-v0.6.32原型闭环优化评估报告.md
docs/decisions/ADR-0001-agent-web-kit采用非侵入式仅对话框集成.md
```

## 4. 升版本规则

小改不升版本。只有信息架构、核心模型、MVP 范围、原型结构或技术路线发生里程碑级变化时，才升级版本号。
