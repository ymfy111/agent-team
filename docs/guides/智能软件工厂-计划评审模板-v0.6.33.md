# 智能软件工厂计划评审模板

> 版本：v0.6.33  
> 日期：2026-05-17  
> 状态：OpenCode 执行前评审模板  
> 目的：借鉴 gstack 的 plan review 思路，把“开工前想清楚”固化为智能软件工厂的任务入口。

---

## 1. 使用时机

以下情况必须先填写本模板：

```text
- 准备创建新的 package / app。
- 准备修改 contracts 类型。
- 准备修改 domain 派生规则。
- 准备新增 mock seed 场景。
- 准备实现跨页面状态流。
- 准备接入外部系统或运行时。
- 准备做 P0b/P1 真实后端、Gateway、OpenCode 集成。
```

## 3. P0a 快速检查

任何 P0a 任务都必须回答：

```text
[ ] 是否保持无后端？
[ ] 是否保持无数据库？
[ ] 是否保持无真实 Gateway？
[ ] 是否保持无真实 OpenCode runtime？
[ ] 是否使用 mock-seed 作为唯一初始数据源？
[ ] 是否通过 domain selectors 派生数据？
[ ] 是否没有暴露 endpointRef/workspaceDir/token？
[ ] 是否没有让普通 Worker 直接对话？
[ ] 是否没有把 routeStatus 写成 status？
```
