# 浏览器自测与交付审查指南

> 版本：v0.6.32  
> 状态：实现验证岗 / 交付审查岗使用浏览器能力的工作规程

---

## 1. 定位

浏览器能力不是普通工具，而是实现验证岗和交付审查岗完成页面自测、交互验证、截图取证和质量门禁的运行时能力。

P0a 阶段不真实接入浏览器工具，只在设计、mock 数据和任务单中预留 BrowserRuntime 模型。P0b 或前端页面开发阶段，默认优先使用 `agent-browser` 作为 BrowserRuntime Provider。

---

## 2. 分层设计

```text
BrowserRuntime
  底层浏览器运行时。
  默认 provider：agent-browser。
  未来可替换为 gstack-browser、Playwright daemon、Browserbase 等。

Browser QA / Acceptance Skill
  上层工作规程。
  定义实现验证岗和交付审查岗如何用浏览器做自测与验收。
```

---

## 3. 实现验证岗如何使用浏览器

实现验证岗使用浏览器属于自测，不等于交付放行。

使用场景：

```text
- 页面启动后检查主导航和关键组件是否可见。
- 任务单状态流转的局部交互验证。
- TDD 完成后的页面行为回归。
- 复现自己发现的问题。
- 为执行回执提供截图或 snapshot 证据。
```

要求：

```text
1. 优先使用 accessibility snapshot / ref，而不是把整页 DOM 塞给大模型。
2. 多步操作尽量 batch，减少命令开销。
3. 保存截图或关键 snapshot 到 artifacts/browser-checks/。
4. 在执行回执中说明浏览器自测范围和结果。
5. 自测通过不代表交付审查通过。
```

---

## 4. 交付审查岗如何使用浏览器

交付审查岗使用浏览器属于独立质量门禁。

使用场景：

```text
- 按任务单验收标准检查页面。
- 验证真实用户路径。
- 检查状态流转是否符合 PRD 和系统设计。
- 截图取证。
- 批量输出问题清单和整改建议。
```

要求：

```text
1. 优先使用独立浏览器 session。
2. 不复用实现验证岗的开发过程状态，除非任务单明确允许。
3. 不做碎片化即时往返。
4. 一次验收输出完整问题清单、风险等级、证据和整改建议。
5. 验收通过后生成 AcceptanceRecord。
6. 验收不通过后生成 ReworkOrder 或 acceptance_fix 类型 WorkOrder。
```

---

## 5. 任务单字段

涉及浏览器的任务单应明确：

```ts
browserRequired?: boolean
browserSessionPolicy?: 'reuse_worker_session' | 'fresh_session' | 'independent_acceptance_session'
evidenceRequired?: boolean
```

建议规则：

```text
实现验证岗自测：reuse_worker_session 或 fresh_session。
交付审查岗验收：independent_acceptance_session。
```

---

## 6. P0a 边界

P0a 只做：

```text
- BrowserRuntimeSession 类型。
- WorkOrder browserRequired / browserSessionPolicy 字段。
- mock 数据中的浏览器 session 状态。
- 原型中的浏览器自测 / 交付审查标签或证据位。
```

P0a 不做：

```text
- 安装 agent-browser。
- 真实调用浏览器。
- 真实截图。
- 真实 QA 自动化。
```
