## 2. 已采纳设计

1. 一个 Worker 对应一个具体 Agent Runtime。
2. 当前 Runtime 实现形态是 OpenCode 工作目录。
3. 使用 RuntimeHost 表达单机或多机运行节点。
4. 使用 WorkerRuntimeBinding 表达 workerId、runtimeHostId、workspaceDir、skillDir、初始化技能版本快照和同步状态。
5. workspaceDir 内的 skill 文件是运行时快照，不是技能资产主数据。
6. P0a 只 mock，不真实写文件；P0b 可单机真实初始化；P1 支持多机调度和跨节点同步。

## 3. 分角色评审

### 产品评审

该设计强化了“数字员工不是静态员工卡，而是可运行智能体实例”的心智，符合智能软件工厂目标。

### 架构评审

RuntimeHost / WorkerRuntimeBinding 把运行节点、工作目录和业务 Worker 解耦，避免把所有工作目录假设在同一台机器。设计合理。

### 数据模型评审

SkillVersion 与 TemplateSkillMapping 仍然是技能真源；workspaceDir 是运行时快照。主数据边界清楚。

### P0a 可执行性评审

P0a 只表达 mock runtimeHosts 和 workerRuntimeBindings，不引入真实文件写入，范围可控。

## 4. SHOULD 建议
