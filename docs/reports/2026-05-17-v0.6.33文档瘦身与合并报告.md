# v0.6.33 文档瘦身与合并报告

> 日期：2026-05-17  
> 状态：当前合并结论  
> 用途：说明为什么可以清空旧 docs 并用 v0.6.33 当前正本替换。

---

## 1. 当前问题

GitHub 当前 docs 目录中存在以下问题：

```text
- specs 下同时存在原始稿、v0.6.29 revised、v0.6.32 revised、参考稿。
- reports 下保留多轮评审过程稿。
- prototypes 下保留多代 HTML 原型和图片资产。
- plans 下仍有早期 v0.4 改动计划。
- 部分 README 当前入口和实际目录内容可能不完全一致。
```

这些材料都曾对探索有价值，但现在会给 OpenCode 带来噪声：它难以判断哪个是当前真源。

---

## 2. 合并策略

### 2.1 specs

合并为 5 个当前正本：

```text
产品需求规格-v0.6.33
系统设计方案-v0.6.33
Mock数据设计-v0.6.33
agent-web-kit集成方案-v0.6.33
P0a工程施工图-v0.6.33
```

旧 PRD / 系统设计 / revised 文档不再保留在当前 docs。

### 2.2 reports

多轮评审报告合并为：

```text
2026-05-17-v0.6.33文档瘦身与合并报告.md
```

评审过程材料由 Git 历史保留。

### 2.3 prototypes

多代 HTML 原型不再作为当前 P0a 开发入口。

处理方式：

```text
- 当前 docs/prototypes/ 只保留 README.md。
- 历史 HTML 原型可通过 Git 历史恢复。
- OpenCode 后续实现以 v0.6.33 施工图为准，不直接读旧 HTML。
```

### 2.4 decisions

保留并升级为 3 个当前 ADR：

```text
ADR-0001 agent-web-kit 非侵入式集成
ADR-0002 monorepo 分层架构
ADR-0003 agent-team 作为业务状态真源
```

### 2.5 guides

合并为：

```text
OpenCode协作指南
计划评审模板
文档维护规则
```

### 2.6 references

只保留当前有明确设计价值的外部参考：

```text
gstack借鉴分析报告
```

agent-web-kit 的完整架构参考不再作为 agent-team 当前正本；相关内容已转化进集成方案和 ADR。

---

## 3. 建议删除 / 不再保留的旧文件类型

```text
docs/specs/*v0.6.29*
docs/specs/*-revised.md
docs/specs/2026-05-13-智能软件工厂产品需求规格.md
docs/specs/2026-05-13-智能软件工厂系统设计方案.md
docs/specs/*reference.md
docs/reports/2026-05-14-*.md
docs/reports/2026-05-15-*.md
docs/plans/2026-04-22-*.md
docs/prototypes/*.html
docs/prototypes/*.png
docs/prototypes/pic/
```

如果担心丢失，可先：

```bash
mv docs docs.backup.$(date +%Y%m%d-%H%M%S)
```

再解压 v0.6.33 当前正本。

---

## 4. 替换后的收益

```text
- OpenCode 读取入口更清楚。
- 每类文档只有一个当前正本。
- 历史探索资料不会再干扰实现。
- v0.6.33 开始具备明确施工图和执行清单。
- 后续文档更新有规则，不再无限增加 revised/final 副本。
```

---

## 5. 风险与处理

风险：旧 HTML 原型不在当前 docs 包中，短期失去视觉对照。

处理：

```text
- 当前阶段以施工图和系统设计为开发入口。
- 如确需视觉对照，从 Git 历史恢复 v0.6.32 HTML 原型。
- 不建议再把多代 HTML 原型放回当前 docs。
```
