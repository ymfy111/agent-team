# 智能软件工厂界面走查报告

> 版本：v0.6.32  
> 日期：2026-05-14  
> 走查范围：`Agent-Team-V2-注册中心-v0.6.32.html`、PRD v0.6.32、设计方案 v0.6.32

## 1. 走查结论

v0.6.32 的主导航心智已经基本稳定：上半部分是运行态，下半部分是软件工厂配置。当前最适合进入 P0a 开发，不建议继续大幅调整一级菜单。

```text
运行态
├── 总览
├── 团队
├── 项目
└── 待决策

工厂配置
├── 岗位
├── 员工
├── 技能
└── 设置
```

本轮建议采用“岗位”作为菜单短名，同时在页面标题和说明中明确“数字员工模板（岗位）”。这样既保留短菜单的易读性，也避免把岗位误解为传统 HR 岗位。

## 2. 页面级走查

### 2.1 左侧导航

**当前状态**：已按运行态 / 工厂配置分区。  
**优化结果**：配置区顺序调整为岗位 → 员工 → 技能 → 设置。

**建议保持**：

- 总览、团队、项目、待决策继续作为运行态入口。
- 岗位、员工、技能、设置继续作为工厂配置入口。
- 不建议把“技能”放回设置页，否则会再次造成“设置只读 vs 技能可编辑”的冲突。

### 2.2 岗位页 / 数字员工模板页

**当前心智**：岗位 = 数字员工模板 = `AgentTemplate`。  
**本轮补强**：页面顶部新增主链路提示。

**后续 UI 建议**：

- 卡片中把“创建数字员工”按钮作为主按钮，而不是弱按钮。
- 模板卡应显示：职责、实例数、在线数、生效技能数、可匹配项目类型。
- 点击“查看技能”应跳到技能页的“岗位技能匹配”并自动选中当前岗位。

### 2.3 员工页 / 数字员工实例页

**当前心智**：员工 = 由岗位模板创建出来的数字员工实例。  
**后续 UI 建议**：

- 员工卡增加“来源岗位”字段，例如：来源模板：测试工程师。
- 员工卡增加“项目匹配”字段，例如：当前项目 / 未匹配。
- 员工详情中应展示完整技能列表，但标明技能来源：通用技能 / 岗位继承。

### 2.4 技能页 / Skill 工作台

**当前心智**：Skill 是一个包含 `SKILL.md` 的能力包。  
**本轮补强**：页面顶部新增工作台说明。

**后续 UI 建议**：

- `SKILL.md` 编辑器应增加“草稿 / 已发布 / 有未发布变更”的状态提示。
- “发布生效”前应弹出影响范围确认。
- 文件树中的 scripts/resources 当前可保留占位，但应在 P1 前定义文件上传和引用规则。

### 2.5 设置页

**当前心智**：设置页只读。  
**本轮修正**：提示文案改为“岗位 / 技能”，不再出现“模板”旧菜单残留。

**建议保持**：

- 设置页只展示运行信息，不承载技能、岗位、员工编辑。
- 开发者配置可以放在小云连接配置面板，不放入全局设置页。

## 3. 需要进入开发任务的 UI 问题

| 编号 | 问题 | 建议优先级 |
|---|---|---:|
| UI-001 | 岗位页创建数字员工仍是占位，需要设计向导 | P0a |
| UI-002 | 员工页缺少“来源岗位 / 当前项目匹配”的强展示 | P0a |
| UI-003 | 技能发布缺少影响范围确认弹窗 | P0a |
| UI-004 | Skill 文件树 scripts/resources 仍是静态占位 | P1 |
| UI-005 | 待决策处理后的回写反馈仍偏弱 | P0b |

## 4. 冻结建议

v0.6.32 可以冻结以下内容：

- 一级菜单结构。
- “岗位”作为 UI 菜单短名。
- `AgentTemplate` 作为技术模型名。
- Skill 以 `SKILL.md` 为核心内容。
- 设置页只读。

下一轮建议不再改主导航，而是进入 P0a 任务开发。
