# v0.6.32 路由状态命名与会话绑定评审修订报告

> 日期：2026-05-15  
> 状态：已修订  
> 范围：AgentRoute / AgentPresence / routeRevision / ConversationRouteBinding / Gateway internal API 安全边界

## 1. 背景

在数字员工运行时设计中，系统引入 RuntimeHost、WorkerRuntimeBinding、AgentRoute 和 AgentPresence。随后对 agent-web-kit Gateway 进行了 Provider / Static / Hybrid 路由解析设计补充。

本轮独立评审发现三类必须收敛的问题：

```text
1. AgentRuntimeRoute.status 与 PublicAgentStatus.routeStatus 命名不一致。
2. routeRevision 只有概念，缺少递增规则和 conversation route snapshot。
3. Gateway internal CRUD / heartbeat 接口安全约束需要前置到接口章节。
```

## 2. 修订内容

### 2.1 统一 routeStatus

AgentRoute 可用性统一使用：

```text
routeStatus = active / disabled / unavailable / error
```

不再新增 `AgentRoute.status` 字段。历史适配层如接收 `status`，进入 domain / selector / DTO 前必须归一化为 `routeStatus`。

### 2.2 增加 routeRevision 递增规则

必须递增 `routeRevision` 的变更：

```text
endpointRef / runtimeHostId / workspaceDir / runtimeType / routeStatus / capabilities
```

不应单独递增 `routeRevision` 的变更：

```text
lastHeartbeatAt / presenceStatus / currentTaskSummary / metrics
```

### 2.3 增加 ConversationRouteBinding

conversation 创建时保存 route snapshot，而不只是 routeRevision。

```ts
type ConversationRouteBinding = {
  conversationId: string
  agentId: string
  routeRevision: number
  runtimeType: 'opencode' | 'mock' | 'custom'
  runtimeHostId?: string
  endpointRef: string
  workspaceDir?: string
  boundAt: string
}
```

### 2.4 前置 internal API 安全硬约束

所有 `/agent-gateway/internal/*` 路由管理和 heartbeat 接口默认关闭，不能由浏览器直接调用，必须使用 service token、mTLS、IP allowlist / CIDR allowlist 中至少一种保护方式。

## 3. 结论

本轮修订后，智能软件工厂侧文档与 agent-web-kit v5 补充设计保持一致。P0a 仍只做 mock 数据与状态展示，不接真实 Gateway；P0b / P1 再逐步实现 runtime-service 与真实 Gateway Provider。
