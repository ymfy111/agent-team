# v0.6.32 原型闭环优化评估报告

> 原型文件：`docs/prototypes/Agent-Team-V2-注册中心-v0.6.32-p0a-flow.html`  
> 优化目标：不做视觉大改，不接真实后端 / Gateway / OpenCode；只增强 P0a 最小闭环可点击、可状态变化、可验收能力。

## 1. 本轮优化结论

本轮原型已经从“功能展示型原型”增强为“P0a 闭环验证型原型”。

现在可以通过原型验证以下主线：

```text
Skill
→ 岗位模板 / AgentTemplate
→ 数字员工 / Worker
→ 团队 / 项目
→ 文档发布
→ 决策处理
→ Activity 回写
```

## 2. 已完成优化

### 2.1 普通员工通过 Leader 联系

- 普通员工入口不再表达为“直接对话”。
- 点击员工沟通入口时，切换到所属团队 Leader。
- Mock 上下文包含 `workerId`、`teamId`、`projectId`、`entry=worker-card`。
- 未入组员工会提示先加入团队。

### 2.2 P0a 演示主线面板

新增右上角“P0a 演示”面板，包含 8 个验收步骤：

1. 发布 Skill
2. 挂载 Skill 到岗位
3. 基于岗位创建员工
4. 员工匹配团队 / 项目
5. 发布项目文档
6. 生成模拟决策
7. 处理决策
8. 查看 Activity 回写

### 2.3 创建数字员工轻量 mock 流程

- 从岗位页点击“创建数字员工”。
- 弹出轻量弹窗，不做复杂向导。
- 创建 Worker 时记录 `templateId`。
- 可以选择加入团队 / 项目职责。
- 加入团队后同步刷新员工、团队、总览和 Activity。

### 2.4 Skill 保存草稿 / 发布生效

- 保存草稿：保留编辑内容，但不 bump version。
- 发布生效：bump version，更新 published content，并写入 Activity。
- 影响范围提示仍基于模板和实例派生计算。

### 2.5 岗位技能关联 / 移除

- 岗位技能匹配页支持关联已有技能。
- 支持移除岗位匹配技能。
- 继续保持 `TemplateSkillMapping` 是唯一主数据。
- `Worker.skills[]` 仍通过 Skill + Mapping + templateId / role 派生。

### 2.6 文档发布增强

- 发布文档后更新文档版本。
- 写入团队 Activity。
- 生成一条模拟 Decision，用于串联文档驱动执行与决策闭环。

### 2.7 决策处理增强

- 决策详情的候选方案可点击。
- 点击后 Decision 状态变为 `resolved`。
- 写入 `resolution`、`handledBy`、`handledAt`。
- 回写团队 Activity。

## 3. 多轮评估过程

### 第 1 轮：产品闭环评估

结论：原型增强方向符合 P0a 主线，但应避免复杂 wizard 和视觉大改。  
处理：创建数字员工改为轻量弹窗；P0a 面板作为验收辅助，不作为正式产品常驻功能。

### 第 2 轮：架构与数据评估

结论：需要防止 HTML mock 破坏数据原则。  
处理：创建 Worker 时记录 `templateId`；技能仍由 `TemplateSkillMapping` 派生；Activity 成为统一回写出口。

### 第 3 轮：QA 验收评估

结论：每个关键动作必须产生可观察状态变化，不能只弹 alert。  
处理：Skill 发布、技能挂载、创建员工、文档发布、决策处理均产生页面状态变化和 Activity。

### 第 4 轮：收敛评估

结论：没有发现新的 P0 阻塞问题。剩余建议均可后置到工程实现或下一轮原型体验优化。

## 4. 当前仍不做的内容

本轮明确不做：

```text
- 真实 agent-web-kit 集成
- 真实 Gateway / OpenCode
- 真实后端 API
- 真实数据库
- 完整权限 / 审计
- Skill 工作台三栏视觉重构
- 岗位页复杂模板-实例关系图
- 生产级文件资源管理
```

## 5. 后续建议

如果后续继续优化原型，建议只做非阻塞增强：

1. Skill 工作台右侧属性栏优化。
2. 岗位页增加模板与实例关系的轻量可视化。
3. 总览页增强配置变更后的运行态投影。
4. 为 P0a 面板增加“一键跳转到相关页面”的辅助能力。

## 6. 评估结论

当前原型已经满足 P0a 闭环演示需求，可以作为后续工程实现的交互参照。
